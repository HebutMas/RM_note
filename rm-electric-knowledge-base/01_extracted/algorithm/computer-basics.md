# 计算机底层知识

> 嵌入式编程必备的底层概念：补码、位操作、无符号整数回卷。

## 补码：计算机怎么做减法

CPU 只有加法电路，没有减法电路。减法通过"加补码"实现。

### 补码的概念

类比时钟：3 点拨到 1 点，可以逆时针拨 2 格（3-2=1），也可以顺时针拨 10 格（3+10=13，表盘只有 12 格，13=1）。在模为 12 的系统里，-2 的补码就是 10。

### 二进制补码计算

对 B 取反 + 1 就是 -B 的补码。例如 4 位下计算 10-7：

```
A = 1010 (10)
B = 0111 (7)
B 取反 = 1000, +1 = 1001 (B 的补码 = -7)
A + B的补码 = 1010 + 1001 = 10011
最高位溢出舍去 → 0011 (3) ✓
```

## 位操作

| 操作 | 符号 | 用途 |
|------|------|------|
| 按位与 | `&` | 清除/检查特定位 |
| 按位或 | `\|` | 设置特定位 |
| 按位异或 | `^` | 翻转特定位 |
| 左移 | `<<` | 乘 2 的幂 |
| 右移 | `>>` | 除 2 的幂 |

嵌入式中最常见：寄存器操作。

```c
GPIOA->MODER |= (0x01 << (pin * 2));   // 设置引脚模式
GPIOA->MODER &= ~(0x03 << (pin * 2));  // 清除引脚模式
```

## 无符号整数回卷

这是 kfifo 和 DWT 时间差计算的核心原理。

### 为什么无符号减法不怕溢出

32 位无符号整数范围 0 ~ 4294967295。假设计数器从 `0xFFFFFFFE` 走 3 步到 `0x00000001`（溢出了）：

```
cnt_now - cnt_last = 0x00000001 - 0xFFFFFFFE
                   = 0x00000001 + 0x100000000(借位) - 0xFFFFFFFE
                   = 3 ✓
```

无符号只有**一个接缝**（最大值→0），借位机制完美弥补，算出来永远是正确的时间差。

### 有符号为什么不行

有符号在 `0111...1`（最大正数）→ `1000...0`（最大负数）处有一个**正负断崖**。跨过这个边界时，真实过了 1 个 tick，但减法算出来是巨大的负数。把 `-252` 塞进 PID 会让电机失控。
