# 环境配置与编译问题处理记录

## 1. PowerShell 任务启动弹窗/闪退问题

**现象**  
在 VS Code 中运行 `Build dji_c` 任务时，终端闪退，或弹出 Windows 的“选择打开方式”对话框，无法正常启动构建流程。

**根因分析**  
系统 `PATH` 中某个目录下存在一个空的 `powershell.exe` 占位（或损坏的链接），且该目录排在真正的 PowerShell 目录前面。当 VS Code 任务管理器使用短名 `"powershell"` 时，系统优先定位到该空目录，导致无法执行真正的 `powershell.exe`。

**排查过程**  
1. 使用 `where.exe powershell` 查看系统定位到的 `powershell.exe` 路径，发现指向一个不存在的空目录。
2. 检查系统环境变量 `PATH`，确认该空目录排在实际 PowerShell 目录（`C:\Windows\System32\WindowsPowerShell\v1.0`）之前。
3. 确认该空目录为无效路径。

**解决方案**  
从系统 `PATH` 中移除该无效的空目录，或直接删除该目录。系统恢复使用真正的 `powershell.exe`。**未修改** `tasks.json` 中的命令路径。

## 2. build 目录误删后的重新配置

**现象**  
修改 `PATH` 或工具链文件后，`compile_commands.json` 仍指向旧编译器路径。删除 `build` 目录后，编译报错，需要重新生成构建系统。

**根因分析**  
CMake 首次配置时将编译器绝对路径写入 `CMakeCache.txt`，后续配置直接复用缓存，不会因 `PATH` 变化而更新。删除 `build` 目录可强制清除缓存，使 CMake 重新探测当前环境。

**操作**  

1. 删除 `build` 目录。
2. 调整系统 `PATH` 或工具链配置。
3.  正常task run
   

## 3. cmake/stm32cubemx 目录误删后的重新生成与代码差异

**现象**  
误删 `cmake/stm32cubemx/` 目录后，编译报链接错误，例如：

- `undefined reference to 'tx_app_rt_pool'`
- 中断服务函数重复定义或缺失
- IWDG 初始化相关错误

**根因分析**  
`cmake/stm32cubemx/` 目录存放 STM32CubeMX 自动生成的硬件初始化代码。误删后需重新生成，但重新生成后的代码与原有代码存在差异，导致业务代码链接失败。

**操作**  

1. 打开工程目录下的 `.ioc` 文件（STM32CubeMX 工程文件）。
2. 在 STM32CubeMX 中点击 **Generate Code**，重新生成 `cmake/stm32cubemx/` 目录下的所有代码文件。

**重新生成后的代码差异（对比原代码）**：

| 文件                 | 差异点                      | 原代码                                 | 新代码                | 影响                        |
| ------------------ | ------------------------ | ----------------------------------- | ------------------ | ------------------------- |
| `app_azure_rtos.c` | `tx_app_rt_pool` 变量      | `static TX_RT_POOL tx_app_rt_pool;` | 去掉 `static`，变为全局定义 | 其他文件引用该变量时需通过 `extern` 声明 |
| `main.c`           | IWDG 初始化调用               | 存在 `MX_IWDG_Init();`                | 该调用被移除             | 若需启用 IWDG 需手动添加           |
| `stm32f4xx_it.c`   | `OTG_FS_IRQHandler` 中断函数 | 存在函数体                               | 该函数被移除             | 转而使用 CherryUSB 的中断处理方式    |

**适配方案**：

- **`tx_app_rt_pool`**：在 `app_azure_rtos.c` 中保留不带 `static` 的定义，在 `main.c` 或其他引用处添加 `extern TX_RT_POOL tx_app_rt_pool;` 声明。
- **IWDG 初始化**：若需启用，在 `main.c` 的硬件初始化部分手动添加 `MX_IWDG_Init();`。
- **OTG_FS 中断**：按 CherryUSB 规范处理 USB 中断，移除原有直接处理逻辑。



## 4. 新板级项目的初始化流程 (以 F103C8T6 移植为例)

### 4.1 初始化调用链

```
main()                                    ← CubeMX 生成的 main.c
  ├─ HAL_Init()
  ├─ SystemClock_Config()
  ├─ MX_GPIO_Init() / MX_CAN_Init() ...   ← CubeMX 外设初始化
  ├─ tx_kernel_enter()                    ← 启动 ThreadX 调度器 (USER CODE 2)
  └─ while(1){}                           ← 调度器接管后不会回到这里

tx_application_define()                   ← ThreadX 回调 (USER CODE 4)
  └─ Robot_Init()
       ├─ tx_byte_pool_create(20KB)       ← 内存池，供 BSP_MEM_ALLOC_WAIT 使用
       ├─ UTILS_Init()                    ← ulog 日志系统
       ├─ BSP_Init()                      ← DWT/LED/BEEP/CAN 总线
       ├─ MODULE_Init()                   ← 各模块初始化 (Offline/Remote/Motor...)
       └─ APP_Init()
            └─ robot_control_init()       ← 应用层，创建 robot_control_thread
                 ├─ gimbal_init()         ← 注册云台电机
                 ├─ chassis_init()        ← 注册底盘电机
                 └─ tx_thread_create()    ← 创建 2ms 控制线程
```

### 4.2 CubeMX 生成后的 main.c 需要改动的区域

以 F103C8T6 为例，CubeMX 生成的是空壳 `main.c`，只调了 `MX_xxx_Init()` 和 `while(1){}`。要在三个 USER CODE 区域插入代码：

**① USER CODE BEGIN Includes** — 添加头文件

```c
/* USER CODE BEGIN Includes */
#include "tx_api.h"
#include "robot_init.h"
/* USER CODE END Includes */
```

**② USER CODE BEGIN 2** — 启动 ThreadX 调度器

```c
  /* USER CODE BEGIN 2 */
  tx_kernel_enter();
  /* USER CODE END 2 */
```

`tx_kernel_enter()` 不会返回，`while(1){}` 仅为占位。

**③ USER CODE BEGIN 4** — 实现 `tx_application_define()`

```c
/* USER CODE BEGIN 4 */
void tx_application_define(void *first_unused_memory)
{
    Robot_Init();
}
/* USER CODE END 4 */
```

ThreadX 调度器启动前会回调此函数，在此处调用 `Robot_Init()` 完成所有初始化。三个修改点可参考 `board/dji_c/Src/main.c` 中已改好的版本。

### 4.3 F103C8T6 内存分配清单

**硬件资源：**
- Flash: 64KB (0x08000000 ~ 0x0800FFFF)
- RAM:  20KB (0x20000000 ~ 0x20004FFF)
- Flash 页: 1KB/页, 共 64 页

**链接脚本 (`STM32F103xx_FLASH.ld`) 默认分配：**
- `.isr_vector`: 0x10C (268B)
- `.text + .rodata`: ~9KB (当前编译结果)
- `_Min_Heap_Size`: 0x200 (512B)
- `_Min_Stack_Size`: 0x400 (1KB)
- `_estack`: 0x20005000 (RAM 顶端)

**当前编译实测：**
```
   text    data     bss     dec     hex
   9508      12    2012   11532   2d0c
```
- Flash 占用: ~9.5KB / 64KB (15%)
- RAM 占用 (链接器可见): ~2KB / 20KB (10%)

**⚠️ 最大风险点：`TX_APP_MEM_POOL_SIZE = 20KB`**

`robot/robot_init.c` 中定义：
```c
#define TX_APP_MEM_POOL_SIZE (20 * 1024)  // 20KB
static UCHAR tx_byte_pool_buffer[TX_APP_MEM_POOL_SIZE];
```

此数组 20KB，F103 总共只有 20KB RAM。**必须缩减**。建议值：

| 用途                | 大小          | 说明                                           |
| ----------------- | ----------- | -------------------------------------------- |
| TX 内存池            | **4KB**     | 供 BSP_MEM_ALLOC_WAIT 动态分配电机/CAN 设备等结构体       |
| 用户 RAM (BSS+DATA) | ~3KB        | 全局变量、CAN 句柄、HAL 句柄                           |
| 线程栈 (累加)          | ~8KB        | 见下方线程栈清单                                     |
| 链接器 Heap+Stack    | 1.5KB       | _Min_Heap_Size(0.5KB) + _Min_Stack_Size(1KB) |
| **总计**            | **~16.5KB** | 预留 3.5KB 余量                                  |

**各线程栈大小（F103 test 配置）：**

| 线程            | 栈大小      | 说明            |
| ------------- | -------- | ------------- |
| CAN RX        | 1024     | CAN 接收线程      |
| CAN TX        | 1024     | CAN 发送线程      |
| Offline 检测    | 1024     | 离线检测任务        |
| Remote 遥控器    | 1024     | SBUS 解码任务     |
| Remote VT 图传  | 1024     | 图传串口任务        |
| Motor 控制      | 1024     | 2ms 电机控制循环    |
| robot_control | 1024     | 应用层 2ms 控制循环  |
| ThreadX 空闲线程  | 512 (默认) | 系统空闲线程        |
| **合计**        | **~8KB** | 不含 Timer/其他模块 |

**缩减建议：**
- 模块栈 1024 → 512：对于简单的 CAN 接收/发送、Offline 检测足够
- 不启用的模块（BMI088/INS/REFEREE/VISION）不占栈空间（条件编译）
- 最终 RAM 占用应控制在 16KB 以内，预留 4KB 余量

### 4.4 Flash 扇区布局

```
0x08000000 ┌──────────────────────┐
           │ 中断向量表 (.isr_vector)│  268B
0x0800010C ├──────────────────────┤
           │ 代码 (.text)          │  ~9KB
0x080024F0 ├──────────────────────┤
           │ 只读数据 (.rodata)     │  44B
0x0800251C ├──────────────────────┤
           │ 初始化数据 (.data LMA)  │  12B
0x08002530 └──────────────────────┘
           │  剩余约 54KB           │  程序代码区
           │                        │
0x0800FC00 ┌──────────────────────┐
           │ 用户参数存储 (最后一页)  │  1KB (页63)
0x0800FFFF └──────────────────────┘
```

用户参数存储在 `board/bsp/FLASH/bsp_flash.c` 中配置为最后一页 `0x0800FC00`（页63，1KB），编译出的 `base.elf` 大小约 9.5KB，远小于 64KB，无需担心空间。
