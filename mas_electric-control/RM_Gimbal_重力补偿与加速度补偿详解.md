## 1. 概述：前馈补偿在云台控制里做什么

### 1.1 一个直觉

把云台 Pitch 轴想象成一根**被电机夹住的单摆**。松开电机（不给电流），枪管会因为重力自己往下垂。要让它稳稳停在某个角度，电机必须持续输出一个**恰好抵消重力矩**的扭矩。

如果只靠 LQR 反馈来干这件事，会发生什么？
- 反馈是**先有误差、再纠正**的：枪口必须先掉下去一点，LQR 才「看到」状态偏差并输出修正力矩拉回来。
- 更要命的是，**纯 LQR 状态反馈没有积分项**（$u=K(x_{ref}-x)$ 只是角度、角速度的比例反馈）。面对重力这种**常值扰动**，它必然停在「反馈力矩恰好等于重力矩」的位置 —— 云台**垂下一个固定的稳态角偏**，怎么调 $K$ 都消不干净（调大只是偏得少些，代价是逼近超调、放大噪声）。

**前馈（feedforward）**的思路正相反：我**提前算出**重力会产生多大的干扰扭矩，直接让电机输出等量反向扭矩把它抵消掉。LQR 反馈只需要处理「算不准的那一点残差」。

```
                 ┌─────────────────────────┐
  x_ref ────►(+)─►│   LQR 反馈  u=K(x_ref−x)  │──┐
              ▲   └─────────────────────────┘  │
              │                                 ▼
      状态 x=[θ,ω]                          (+)──► 电机总扭矩 ──► 云台
              │                                 ▲
              └──── 编码器/IMU 反馈 ◄────────┐   │
                                            │   │
   角度/姿态 ──►┌──────────────┐  前馈扭矩   │   │
              │ 重力/加速度补偿  │───────────────┘
              └──────────────┘
```

前馈项和 LQR 反馈项**相加**送给电机。前馈负责「大方向对」，LQR 负责「精确收敛」。

### 1.2 $u$ 里的重力从哪来（不加前馈时）

一个很自然的困惑：「我 LQR 脚本建模时，$u$ 就是电机转矩、模型里 $\dot x = Ax + Bu$ 根本没有重力项，那我没加重力补偿之前，输出的 $u$ 是不是其实已经含了重力力矩？」

**答案：是的，$u$ 里早就含着重力那一份——物理逼的，躲不掉。** 关键要把两个不同的 $u$ 分开：

**① 建模时的 $u$（用来算 $K$ 的）——不含重力，也本就不该含。**
脚本里的模型是纯的 $J\ddot\theta = u$，重力被丢掉了。这是对的：重力是**扰动**，不是被控对象 $(A,B)$ 的一部分，$K$ 只对着这个干净模型算最优，不该掺重力。

**② 运行时电机真正输出的 $u$——必含重力。**
真实动力学其实是
$$
J\ddot\theta = u - \tau_g(\theta), \qquad \tau_g = mgr\cos(\theta+\gamma)
$$
云台要静止（$\ddot\theta=\dot\theta=0$），物理逼出 $u_{ss} = \tau_g$——不输出这份力矩枪管就下坠。

**纯 LQR 是怎么凑出这份重力力矩的？** 反馈律 $u=K(x_{ref}-x)$ 根本不知道重力，它只能**靠状态误差**来生产。设 $K=[k_1,\ k_2]$，稳态角速度为 0：
$$
u_{ss}=k_1(\theta_{ref}-\theta_{ss})=\tau_g
\;\Longrightarrow\;
\underbrace{\theta_{ref}-\theta_{ss}}_{\text{下垂角}}=\frac{\tau_g}{k_1}
$$
**云台先垂下 $\tau_g/k_1$ 一个角，让「误差×增益」恰好等于重力矩，才凑齐托枪的力。** 那个稳态下垂不是 bug，是纯 LQR **生产重力力矩的唯一手段**。

**那加前馈改了什么？——不是「加力」，是「换来源」：**
$$
\text{加前馈前：}\ u_{ss}=\underbrace{K(x_{ref}-x_{ss})}_{=\tau_g,\ \text{靠下垂凑}}
\qquad
\text{加前馈后：}\ u_{ss}=\underbrace{\tau_g}_{\text{前馈直接给}}+\underbrace{K(x_{ref}-x_{ss})}_{\approx 0}
$$
两种情况电机输出的**总力矩几乎一样，都 $\approx\tau_g$**。前馈没凭空多出力矩，只是把「重力那一份」从**反馈（靠误差挣来）**挪到**前馈（明码标价直接给）**；反馈不用再靠误差挣这份力矩，误差就归零 → **下垂消失**。这也正面回答了「加前馈会不会力矩翻倍、云台变硬」——不会，是换来源不是加力。

> **为什么前馈能干净接管？** 重力是**匹配扰动（matched disturbance）**：真实方程 $\dot x=Ax+Bu-B\tau_g$ 里，重力 $-B\tau_g$ 和控制量 $u$ 走**同一个输入通道 $B$**，活在同一空间。所以一个 $u$ 分量能和重力**逐项对消**——这既解释了「$u$ 为何必然能含重力」，也解释了「前馈为何能完美抵消它」。

> 一句话速记：**建模的 $u$ 不含重力（重力是扰动不是对象）；但运行时的 $u$ 必含重力（物理逼的），纯 LQR 靠下垂 $\Delta\theta=\tau_g/k_1$ 来凑。加前馈不改总力矩，只把重力那份从「反馈靠误差挣」换成「前馈直接给」，于是下垂归零。重力是匹配扰动、和 $u$ 同通道，所以能对消。**

### 1.3 三个层次

本文按建模精度从粗到细分三层，工程上按需选用：

| 层次         | 处理的物理量          | 假设           | RM 适用场景         |
| :--------- | :-------------- | :----------- | :-------------- |
| **简易重力补偿** | 只补 Pitch 轴重力矩   | 底盘水平、Yaw 轴竖直 | 绝大多数步兵/英雄，够用    |
| **一般重力补偿** | Yaw+Pitch 双轴重力矩 | 底盘可有倾角       | 平衡步兵、爬坡、工程      |
| **加速度补偿**  | 重力矩 + 运动惯性力矩    | 无（含底盘加减速）    | 高机动底盘的极致跟随，力控练习 |

> 一句话速记：**重力补偿是「静态站得住」，加速度补偿是「动态跟得上」，后者是前者的超集。**

## 2. 坐标系与符号约定

> 后文所有推导都建立在这套约定上。这一节是「字典」，读不懂公式时回来查。

### 2.1 四个坐标系

| 记号 | 名称 | 含义 |
|:----|:----|:----|
| $O$ | Odometry | 大地坐标系（惯性系），重力方向固定 |
| $C$ | Chassis | 底盘坐标系，底盘 IMU 固连于此 |
| $Y$ | Yaw | 云台 Yaw 电机转过后的坐标系 |
| $P$ | Pitch | 云台 Pitch 电机转过后的坐标系，云台 IMU 固连于此 |

层级关系（从大地到枪口）：$O \to C \to Y \to P$。

### 2.2 上下标记法（关键，务必看懂）

$${}^{A}_{C}\vec{q}_{B}$$

表示：**在 $A$ 坐标系下观察**，$B$ 相对于 $C$ 的某个矢量。若参考系 $C$ 与观察系 $A$ 相同，则省略左下标 $C$。

- 例：${}^{C}_{O}\vec{\omega}_{C}$ = 在 $C$ 系下表达的、$C$ 相对 $O$ 的角速度（就是底盘 IMU 直接读到的角速度）。
- 例：${}^{P}\vec{g}$ = 在 $P$ 系下表达的重力加速度矢量。

旋转矩阵简记：${}^{A}B$ 表示把 $B$ 系下的矢量转到 $A$ 系下的旋转矩阵（即 $\vec{v}^{A} = {}^{A}B \cdot \vec{v}^{B}$）。

> **核心纪律**：只有在**同一坐标系下**的矢量才能直接加减 / 叉乘。跨系必先用旋转矩阵转换。这是所有推导不出错的前提。

### 2.3 电机角度约定（右手定则）

| 量 | 定义 | 正方向 |
|:----|:----|:----|
| $\alpha$ | Yaw 电机角度，角速度 $\dot\alpha$ | 云台**左转**为正（绕 $z$ 轴） |
| $\beta$ | Pitch 电机角度，角速度 $\dot\beta$ | 云台**低头**为正（绕 $y$ 轴） |
| $\gamma$ | 重心偏置角 | Pitch 重心相对枪线的固定夹角 |

> 记号约定：物理量上方一个点表示对时间求一阶导（速度/角速度），两个点表示二阶导（加速度/角加速度）。如 $\dot\alpha$ 是 Yaw 角速度，$\ddot\alpha$ 是 Yaw 角加速度。

### 2.4 质量与惯量建模

- **Pitch 轴固连机构**（枪管、Pitch 电机转子、拨弹等）：质量 $m$ 可由称重或工程制图软件（SolidWorks 等）直接给出。这是补偿的主要对象。
- **Yaw 轴固连机构**：质量近似取 **0**（Yaw 轴通常竖直，重力对它几乎不产生绕 Yaw 的力矩；加速度补偿时才细究）。
- **惯量矩阵**：把 Pitch 负载粗建模为长度 $l$、质量 $m$ 均匀、重心居中的**杆**（不是圆柱、不是长方体）。当杆沿 $x$ 轴时，绕重心的惯量矩阵为

$$
I = \begin{pmatrix} 0 & 0 & 0 \\ 0 & \dfrac{ml^2}{12} & 0 \\ 0 & 0 & \dfrac{ml^2}{12} \end{pmatrix}
$$

## 3. 简易重力补偿（水平地面）

### 3.1 场景与直觉

**假设**：底盘停在水平地面上，Yaw 轴竖直、Pitch 轴水平。

在这种情况下，重力只对 **Pitch 轴**产生力矩（Yaw 轴竖直，重力沿 Yaw 轴方向，对绕 Yaw 的转动没有力臂）。所以只需补 Pitch 一个轴。

**为什么不能当质点算？** 因为我们分析的是**转动**。云台 Pitch 电机角度一般以「枪口水平」定为 $\beta=0$。但当 $\beta=0$（云台水平）时，Pitch 负载的**重心**与 Yaw 轴原点的连线**不一定水平** —— 重心可能偏在枪线的上方或下方。这个固定偏差就是重心偏置角 $\gamma$。

```
         枪线方向 (β 定义的方向)
        ─────────────────►
       Y原点 •
              \  γ
               \ ← 重心与枪线的固定夹角
                • 重心 (质量 m 集中于此)
                  \  r = |Y原点→重心|
```

### 3.2 核心公式

设重心相对枪线的旋转角为 $\gamma$，当前 Pitch 电机角为 $\beta$，则重力对 Pitch 轴产生的力矩大小：

$$
t = m g r \cos(\beta + \gamma)
\tag{3.1}
$$

- $m$：Pitch 轴固连负载质量
- $g$：重力加速度（$\approx 9.8\,\mathrm{m/s^2}$）
- $r$：Pitch 重心到 **Pitch 轴**的垂直距离（力臂长度）。常规云台 Yaw / Pitch 两轴相交于云台中心，Pitch 轴过 Yaw 原点，故从 Yaw 原点量得的距离与之相等；若两轴不相交（Pitch 轴相对 Yaw 有偏置），必须以 Pitch 轴为准
- $\beta+\gamma$：重心连线与水平面的实际夹角

**为什么是 $\cos$？** 重力矩 = 力 × 水平力臂。重力 $mg$ 竖直向下，力臂是重心到转轴的**水平投影**，即 $r\cos(\beta+\gamma)$。当重心连线水平（$\beta+\gamma=0$）时 $\cos=1$，力矩最大；当重心连线竖直时 $\cos=0$，力矩为 0（此时重力过转轴，无力臂）。这完全符合单摆直觉。

补偿时电机输出**反向**力矩 $q$ 抵消它：

$$
q + k\cos(\beta + \gamma) = 0, \qquad k = mgr
\tag{3.2}
$$

只要标定出 $k$ 和 $\gamma$ 两个参数，简易重力补偿就完成了。

### 3.3 两种确定参数的方法

#### 方法一：测量法

直接测物理量：
- 用秤称出 Pitch 负载质量 $m$，用**悬挂法**（两次悬挂求重心交点）确定重心位置 → 得 $r$、$\gamma$。
- 或者直接找机械要图纸，SolidWorks 等软件能算出质量和重心坐标。

**缺点**：需要在机械半成品阶段标定；后期加走线、附件会改变重心；图纸零件常只有尺寸没有质量。

#### 方法二：标定法（推荐）

把 $k, \gamma$ 当作待拟合参数，通过实验数据反解：

1. 给 Pitch 电机施加不同的**恒定输出扭矩** $q$（如 $0.2\,\mathrm{Nm}, 0.5\,\mathrm{Nm}$…）。
2. 手拨 Pitch 轴，观察在每个 $q$ 下云台能**稳定停住**的角度 $\beta$（力矩平衡点）。
3. 得到一系列 $(q, \beta)$ 映射对。
4. 代入 $q = -k\cos(\beta+\gamma)$，用**最小二乘法**拟合 $k, \gamma$。

**为什么推荐**：通用性强、机械成型后即可标定、自动包含了走线等附加质量的影响。这也是和电机功率建模里同一套最小二乘思路。

拟合展开技巧：$\cos(\beta+\gamma) = \cos\beta\cos\gamma - \sin\beta\sin\gamma$，令 $a=k\cos\gamma,\ b=k\sin\gamma$，则 $q = -a\cos\beta + b\sin\beta$ 对 $(a,b)$ 是**线性**的，可直接线性最小二乘，再由 $k=\sqrt{a^2+b^2},\ \gamma=\operatorname{atan2}(b,a)$ 还原。

#### 变体：闭环 ref 扫角标定（有 LQR 时推荐）

上面的恒扭矩法是**开环**给力矩、读平衡角。如果你已经有能跑的 LQR，可以反过来：**闭环给不同的目标角 $\theta_{ref}$，等云台停稳，记录（稳态实际角 $\theta_{ss}$，稳态输出力矩 $u_{ss}$）**，用同一个最小二乘拟合 $k,\gamma$。

**为什么可行——标定只认物理平衡点，不认控制器。** 任何控制器停在稳态（$\ddot\theta=\dot\theta=0$）时，动力学 $J\ddot\theta=u-\tau_g(\theta)$ 直接给出：
$$
u_{ss} = \tau_g(\theta_{ss}) = mgr\cos(\theta_{ss}+\gamma)
$$
这个式子**与控制器是 P/PD/LQR/PID 无关，是纯物理**。所以无论你用「开环给力矩读角」还是「闭环给角读力矩」，采到的点都落在同一条曲线 $u=mgr\cos(\theta+\gamma)$ 上，拟合数学完全一样。

**闭环给 ref 的三个优势**：

1. **全程稳定、不用手扶**。恒扭矩开环时，部分区间的平衡点不稳（原方法才要「手拨探测能稳的范围」）；闭环 LQR 把**每个**命令角都镇住，可让 ref 从最低扫到最高、密集安全地采满整个 Pitch 行程。
2. **刚度再大也测得到力矩**。别担心「LQR 硬、稳态误差趋零就没信号」——托住角度 $\theta_{ss}$ 所需的力矩由物理决定、**与 $k_1$ 无关**；刚度只影响下垂角（实际角离目标多远），不影响力矩大小。水平位仍需输出约 $mgr$ 的大力矩，信号很强。
3. **能双向逼近抵消摩擦**。静摩擦让云台停在一个死区里（$|\tau_g-u|<f_{fric}$ 就不动），给拟合带来散点。闭环下对每个 ref **从上、从下各逼近一次取平均**，库仑摩擦方向相反、直接抵消。

**三个必须避开的坑**：

1. **cos 里用实际角 $\theta_{ss}$，不是命令角 $\theta_{ref}$**。重力矩只认实际姿态——每个点的角度必须取编码器/IMU 实测的 $\theta_{ss}$；若图省事拿 $(u_{ss},\theta_{ref})$ 配对，就把下垂角当误差塞进了 cos，标出的 $\gamma$ 会偏。
2. **直接读「真实总输出力矩」，别用 $k_1\cdot$误差反推**。最稳做法是 log 电机总力矩指令（DM MIT 模式即力矩；DJI 电流环用 $u=I\cdot K_t$ 换算）。为何不用 $u_{ss}=k_1(\theta_{ref}-\theta_{ss})$ 算？因为那要求你**已准确知道 $k_1$**，而 $k_1$ 真机值本身就存疑（仿真 $\sqrt{q_1}$ vs 实际有离散化/单位坑），用不确定量去标定另一个是循环论证；直接读输出力矩则**根本不需要 $k_1$**。
3. **关掉前馈、注意积分项、等停稳**。① 前馈必须关，否则 $u_{ss}$ 混了前馈、标的是残差不是重力；② 若开了 LQI（积分），稳态误差归零，「$k_1\cdot$误差」彻底失效，但**直接读总输出力矩仍成立**（$u_{ss}=\tau_g$ 是物理，与有无积分无关）——所以坑 2 在这里更是刚需；③ 必须等 $\dot\theta\approx0$ 真正停稳再采，否则动态惯性力矩污染数据。

> 一句话速记：**给不同 ref 拟合完全可行——标定只认物理平衡点 $u_{ss}=mgr\cos(\theta_{ss}+\gamma)$，与控制器无关。闭环比开环恒扭矩更安全、可扫满行程、能双向抵消摩擦。三条铁律：用实测角 $\theta_{ss}$、直接读总输出力矩（别拿 $k_1\times$误差反推）、关前馈并等停稳。**

#### 视觉验证：随动随停

很多开源视频里 Pitch 轴有个惊艳效果——**用手推到任意角度松手，它就停在那儿不动，像失重一样**。这个「随动随停」正是重力补偿拟合质量的**连续视觉验证**，比上面离散测点更直观。

> ⚠️ **先澄清一个常见误解**：随停效果**与 LQR 无关，纯粹是前馈在起作用**。「推哪停哪」的数学条件是 $u_{ff}(\theta)=\tau_g(\theta)$ 对**所有角度都成立** → 每个角度净力矩都为零 → 云台在整个行程里处处**中性平衡**。这只依赖前馈拟合得准。而 LQR 的位置反馈 $K(x_{ref}-x)$ 若开着，你一推离目标它就**弹回那一个固定角**（定角保持），恰恰**破坏**随停。所以看到随停，就说明该演示**位置反馈是关掉的、画面里只有前馈**；看到弹回固定角，那才是 LQR 反馈在干活。两者互斥，不会同屏。（运行时二者是协同的：前馈托重力、LQR 保角度；演示只是把位置保持关掉，单独展示前馈。）

**操作**：只开前馈、关掉位置保持（可留角速度阻尼项让它不乱晃），全程手拨 Pitch 轴，看它是否**处处都停得住**。由表现即可反推拟合质量：

| 表现              | 说明                         | 参数问题       |
| :-------------- | :------------------------- | :--------- |
| 整个行程都能停住，手感均匀失重 | $u_{ff}\approx\tau_g$ 处处成立 | 标得准 ✓      |
| 往下压它会自己往上抬      | 前馈托过头                      | $k$ 偏大     |
| 某些角度仍往下垂        | 前馈托不住                      | $k$ 偏小     |
| 偏离水平时一边沉一边飘、不对称 | 相位错位                       | $\gamma$ 错 |

哪个角度飘，就说明那一段的 $k,\gamma$ 拟合有偏差——这就是作者在视频里肉眼校验拟合的方式。

> 一句话速记：**随停 = 前馈让云台处处中性平衡，是纯前馈拟合质量的展示；LQR 位置保持只会把它弹回固定角、反而破坏随停。看到随停 → 位置反馈已关，只有前馈在托重力。哪个角飘，哪段就没标准。**

### 3.4 关于 γ 的讨论

- **$\gamma$ 有必要吗？** 通常 $\gamma$ 很小，可以忽略（直接用 $t=k\cos\beta$）。但标定了会更准。
- **前馈只要大方向对就有用**：即使假设不完全成立（比如轻微斜坡），前馈给个大致正确的扭矩，把大部分重力矩抵消掉，剩下的小残差交给 LQR 反馈收拾，仍然比完全没有前馈好得多。这就是前馈+反馈互补的意义。
- **注意 LQR 无积分、残差不会归零**：纯 LQR 状态反馈没有积分项，前馈没抵消干净的那部分残差会留下一个（更小的）稳态角偏，反馈只能压小、不能清零。若要连这点残差也消除，需给 LQR 做积分增广（LQI，见 §7.4），而不是指望比例反馈把它磨没。

> 一句话速记：**Pitch 重力矩 $= mgr\cos(\beta+\gamma)$，电机反着出力；$k=mgr$ 和 $\gamma$ 用「给定扭矩测平衡角 + 最小二乘」标出来。**

## 4. 一般重力补偿（任意底盘姿态）

### 4.1 为什么斜坡上简易补偿会失效

简易补偿假设「Yaw 轴竖直、重力沿 Yaw 轴」。一旦底盘上斜坡，这个前提就崩了：

- 重力方向在**底盘系下不再竖直**，它会分出一个绕 Yaw 轴的分量 → **Yaw 轴也需要补偿**。
- Pitch 的重力矩也不能再简单用 $\cos(\beta+\gamma)$ 算，因为底盘倾角改变了重力在 Pitch 系下的投影。

解决思路：**别再猜重力方向，而是把大地系下已知的重力矢量 $(0,0,-g)$ 一路旋转到云台各轴的坐标系里**，再算力矩。这需要一条完整的旋转矩阵链。

### 4.2 P 系原点的重新定义

从本节起，把坐标系 $P$ 的原点**放在 Pitch 重心上**（而非电机轴心），这样重力作用点就是原点，力矩计算更干净：
- $x$ 轴：Yaw 原点指向 Pitch 重心的方向
- $y$ 轴：保持 Pitch 转轴方向
- $z$ 轴：右手定则补齐

### 4.3 旋转矩阵链

要把大地重力 ${}^{O}\vec{g}=(0,0,-g)^T$ 转到 $P$ 系，逐级左乘：

$$
{}^{P}\vec{g} = {}^{P}Y \cdot {}^{Y}C \cdot {}^{C}O \cdot {}^{O}\vec{g}
\tag{4.1}
$$

| 旋转矩阵 | 来源 |
|:----|:----|
| ${}^{C}O$ | 底盘 IMU 读出欧拉角后算得（描述底盘在大地系的倾斜姿态） |
| ${}^{Y}C$ | 由 Yaw 电机角度 $\alpha$ 算得，即 $\mathrm{Yaw}^T(\alpha)$ |
| ${}^{P}Y$ | 由 Pitch 电机角度算得，即 $\mathrm{Pitch}^T(\beta+\gamma)$ |

> 提示：${}^{Y}C = \mathrm{Yaw}^T(\alpha)$ 中的转置，是因为 $\mathrm{Yaw}(\alpha)$ 通常定义为「把 $Y$ 系矢量转到 $C$ 系」的矩阵，而这里要反向（$C\to Y$）。旋转矩阵正交，逆等于转置。

同理只转到 $Y$ 系（算 Yaw 力矩时用）：

$$
{}^{Y}\vec{g} = {}^{Y}C \cdot {}^{C}O \cdot {}^{O}\vec{g}
\tag{4.2}
$$

### 4.4 力矩 = 力臂 × 重力（叉乘形式）

重力对某转轴的力矩，用叉乘表达。记 $\vec{r}$ 为**从转轴指向重力作用点（重心）**的位置矢量，则重力矩 $\vec{n} = \vec{r} \times (m\vec{g})$。讲义里写成 $\vec{n} = -m\vec{g}\times\vec{r}$，两者等价（叉乘反交换律 $\vec{a}\times\vec{b}=-\vec{b}\times\vec{a}$）。

**Pitch 轴力矩**（在 $P$ 系，力臂是重心指向 Yaw 原点）：

$$
{}^{P}\vec{n}_P = -m\, {}^{P}\vec{g} \times {}^{P}\vec{r}_Y
\tag{4.3}
$$

**Yaw 轴力矩**（在 $Y$ 系，力臂是重心指向 Chassis 原点）：

$$
{}^{Y}\vec{n}_P = -m\, {}^{Y}\vec{g} \times {}^{Y}_{P}\vec{r}_C
\tag{4.4}
$$

其中力臂的构成：
- ${}^{P}\vec{r}_Y$：$P$ 系下，Pitch 重心指向 Yaw 原点的矢量，形如 $(-r, 0, 0)$（沿 $x$ 反向）。
- ${}^{Y}_{P}\vec{r}_C$：$Y$ 系下，Pitch 重心指向 Chassis 原点的矢量。它由两段拼成：

$$
{}^{Y}_{P}\vec{r}_C = {}^{Y}P \cdot {}^{P}\vec{r}_Y + {}^{Y}\vec{r}_C
$$

- 第一段 ${}^{Y}P\cdot{}^{P}\vec{r}_Y$：重心→Yaw 原点（转到 $Y$ 系）
- 第二段 ${}^{Y}\vec{r}_C$：Yaw 原点→Chassis 原点，形如 $(0,0,-h)$（Yaw 轴有一段竖直高度）

### 4.5 投影到电机轴

力矩是三维矢量，但电机只能绕自己那一根轴出力。所以把力矩**投影**到电机转轴的单位方向上：

$$
n_p = {}^{P}\vec{n}_P^{\,T}\, \vec{e}_y, \qquad
n_y = {}^{Y}\vec{n}_P^{\,T}\, \vec{e}_z
\tag{4.5}
$$

- Pitch 电机绕 $y$ 轴 → 投影到 $\vec{e}_y=(0,1,0)^T$
- Yaw 电机绕 $z$ 轴 → 投影到 $\vec{e}_z=(0,0,1)^T$

$n_p, n_y$ 就是两个电机各自需要输出的重力补偿前馈扭矩。

> **与简易补偿的一致性检查**：当底盘水平（${}^{C}O=I$）时，${}^{P}\vec g$ 退化，$(4.3)$ 投影到 $y$ 轴后正好还原成 $mgr\cos(\beta+\gamma)$，且 $(4.4)$ 投影到 $z$ 轴为 0（Yaw 无需补偿）。所以一般补偿是简易补偿的推广，两者在水平工况下自洽。

> 一句话速记：**把大地重力 $(0,0,-g)$ 用旋转矩阵链转进 Yaw/Pitch 系，各自 $\vec r\times m\vec g$ 求力矩，再投影到电机轴。斜坡上 Yaw 也得补。**

## 5. 加速度补偿（进阶）

> ⚠️ 这一部分硬核，作为力控思想的训练。工程上简易/一般重力补偿已足够 RM 电控使用。看不懂可跳到第 6 节。

### 5.1 核心直觉：加速度补偿是重力补偿的超集

六轴 IMU 读到的加速度**天然包含重力**。举例：

- 一个良好第一人称跟随的底盘以 $1\,\mathrm{m/s^2}$ 向前加速。
- 底盘 IMU 反馈的加速度约为 $(1, 0, 9.8)$：
  - $x$ 分量 $1$ → 底盘前冲，会让 Pitch 产生「上抬」的惯性冲动（枪口相对被甩）。
  - $z$ 分量 $9.8$ → 就是重力本身。

既然 IMU 把重力和运动加速度**混在一个矢量里**，那我们只要把「重力加速度」换成「重力 + 运动加速度」的合成矢量，套同一套力矩公式，就同时补偿了重力和惯性力。**所以加速度补偿 ⊃ 重力补偿。**

代价：需要底盘的加速度、角加速度，以及云台各级的速度/加速度递推 —— 数学量陡增。

### 5.2 数学工具箱

#### 5.2.1 叉乘的矩阵形式

两个列向量叉乘可写成「反对称矩阵 × 向量」：

$$
\vec{a}\times\vec{b} =
\begin{pmatrix} 0 & -a_z & a_y \\ a_z & 0 & -a_x \\ -a_y & a_x & 0 \end{pmatrix}
\begin{pmatrix} b_x \\ b_y \\ b_z \end{pmatrix}
$$

于是可把「$\vec a\times$」当成一个矩阵算子 $A_\times$ 参与运算：

$$
\vec{a}\times \;=\; A_\times = \begin{pmatrix} 0 & -a_z & a_y \\ a_z & 0 & -a_x \\ -a_y & a_x & 0 \end{pmatrix}
$$

两个推论：
- $A_\times$ 是**反对称矩阵**（$A_\times + A_\times^T = O$），3 阶反对称阵有 3 个自由度，恰由 $\vec a$ 的三个分量唯一确定。
- 叉乘遇旋转矩阵 $R$ 可「穿透」：$R(\vec a\times\vec b) = (R\vec a)\times(R\vec b)$。（旋转不改变叉乘的几何关系，故成立。）

#### 5.2.2 旋转矩阵对时间求导

设 $R={}^{A}B$，$B$ 相对 $A$ 的角速度为 $\vec\omega$，$B$ 系下固定点 $\vec q$（相对 $B$ 不动）。该点在 $A$ 系的速度有两种写法：

由求导：$\dfrac{d(R\vec q)}{dt} = R\dot{\vec q} + \dot R\vec q = \dot R\vec q$（因 $\dot{\vec q}=0$）。

由角速度：$\vec\omega\times(R\vec q)$。

两者相等，对任意 $\vec q$ 成立，故得到关键恒等式：

$$
\boxed{\dfrac{dR}{dt} = \vec\omega\times R}
\tag{5.1}
$$

形式化推广（不证）：$\dfrac{dI}{dt} = \vec\omega\times I$，以及 $\dfrac{d(\vec\omega\times)}{dt} = \vec\alpha\times$（$\vec\alpha$ 为角加速度）。

#### 5.2.3 位置 / 速度 / 加速度的坐标系间递推

设两坐标系 $A\to B$，$B$ 系下有物体 $Q$。已知 $B$ 相对 $A$ 的坐标 ${}^{A}\vec b$、速度 ${}^{A}\vec v_B$、角速度 ${}^{A}\vec\omega_B$、加速度、角加速度；以及 $Q$ 相对 $B$ 的对应量。则从 $A$ 系看 $Q$ 的运动状态逐级递推：

**位置**：
$$
{}^{A}\vec q = {}^{A}\vec b + {}^{A}B\, {}^{B}\vec q
$$

**速度**（对位置求导，用到 $(5.1)$）：
$$
{}^{A}\vec v_Q = {}^{A}\vec v_B + {}^{A}B\,{}^{B}\vec v_Q + {}^{A}\vec\omega_B\times {}^{A}B\,{}^{B}\vec q
\tag{5.2}
$$
> 推导要点：$\frac{d}{dt}({}^{A}B\,{}^{B}\vec q) = {}^{A}B\,\dot{{}^{B}\vec q} + \dot{{}^{A}B}\,{}^{B}\vec q$，第二项用 $\dot R=\vec\omega\times R$ 变成叉乘项。三项分别是「牵连平动 + 相对速度 + 牵连转动」。

**角速度**（直接叠加）：
$$
{}^{A}\vec\omega_Q = {}^{A}\vec\omega_B + {}^{A}B\,{}^{B}\vec\omega_Q
$$

**加速度**（对速度再求导，含科氏项与向心项）：
$$
{}^{A}\vec a_Q = {}^{A}\vec a_B + {}^{A}B\,{}^{B}\vec a_Q
+ 2\,{}^{A}\vec\omega_B\times {}^{A}B\,{}^{B}\vec v_Q
+ {}^{A}\vec\alpha_B\times {}^{A}B\,{}^{B}\vec q
+ {}^{A}\vec\omega_B\times {}^{A}\vec\omega_B\times {}^{A}B\,{}^{B}\vec q
\tag{5.3}
$$
> 五项含义：牵连平动加速度、相对加速度、**科氏加速度**（$2\vec\omega\times\vec v$）、角加速度切向项、**向心加速度**（$\vec\omega\times\vec\omega\times\vec r$）。这就是理论力学里的加速度合成定理。

**角加速度**：
$$
{}^{A}\vec\alpha_Q = {}^{A}\vec\alpha_B + {}^{A}B\,{}^{B}\vec\alpha_Q + {}^{A}\vec\omega_B\times {}^{A}B\,{}^{B}\vec\omega_Q
$$

#### 5.2.4 力与扭矩的连杆递归

把云台看作串接连杆：基座（底盘）编号 $0$，第 $i$ 个连杆编号 $i$。已知末端杆需要的力/扭矩，可**从末端递归回基座**。设杆 $i$ 受合外力 ${}^{i}\vec\phi_i$、合外扭矩 ${}^{i}\vec\tau_i$，末端需提供的力 ${}^{i}\vec f_i$、扭矩 ${}^{i}\vec n_i$，相邻杆旋转矩阵记 ${}^{i-1}_{i}R$：

**力的递归**：
$$
{}^{i-1}\vec f_{i-1} = {}^{i-1}\vec\phi_{i-1} + {}^{i-1}_{i}R\, {}^{i}\vec\phi_i
$$

**扭矩的递归**（多了两个力臂叉乘项）：
$$
{}^{i-1}\vec n_{i-1} = {}^{i-1}\vec\tau_{i-1} + {}^{i-1}_{i}R\, {}^{i}\vec\tau_i
+ {}^{i-1}\vec i\times {}^{i-1}\vec\phi_{i-1}
+ {}^{i-1}\vec i\times {}^{i-1}_{i}R\, {}^{i}\vec\phi_i
$$
> 这是 Newton-Euler 逆动力学递归的标准形式：每级把本级合外力/矩，加上下级传来的力/矩（转到本级系并考虑力臂）。

#### 5.2.5 Newton 与 Euler 公式

作用点在重心时：

- **Newton**（平动）：$\vec f = \dfrac{d}{dt}(m\vec v) = m\vec a$
- **Euler**（转动）：$\vec n = \dfrac{d}{dt}(I\vec\omega) = I\vec\alpha + \vec\omega\times I\vec\omega$
  $$\tag{5.4}$$
  > Euler 公式第二项 $\vec\omega\times I\vec\omega$ 是**陀螺力矩**，来自惯量矩阵随姿态旋转（$\dot I=\vec\omega\times I$）。低速时可忽略，高速旋转时不可省。

### 5.3 加速度补偿完整计算流程

思路：**从底盘出发，逐级递推 Yaw、Pitch 的运动状态 → 代入 Newton/Euler 求合外力矩 → 用连杆递归推回电机轴。**

#### 步骤 0：采集底盘运动状态

| 量 | 来源 |
|:----|:----|
| 底盘速度 ${}^{C}_{O}\vec v_C$ | 底盘电机运动学逆解算 |
| 底盘角速度 ${}^{C}_{O}\vec\omega_C$ | 底盘 IMU 直接读 |
| 底盘加速度 ${}^{C}_{O}\vec a_C$ | 底盘 IMU 直接读 |
| 底盘角加速度 ${}^{C}_{O}\vec\alpha_C$ | 角速度**差分** + 低通滤波（差分放大噪声，必须滤波） |

> **简化**：底盘的位置和速度对最终力矩结果无影响（力矩只关心加速度层级），云台各级的相对位置由机械图纸固定、相对速度为 0（电机角速度单独算）。可舍去无关量。

#### 步骤 1：递推云台 Yaw 的运动状态

用 $(5.2)(5.3)$ 令 $A=O, B=C, Q=Y$，${}^{C}\vec y$ 是 $C$ 原点指向 $Y$ 原点的矢量：

$$
{}^{Y}_{O}\vec v_Y = {}^{Y}C\left({}^{C}_{O}\vec v_C + {}^{C}_{O}\vec\omega_C\times {}^{C}\vec y\right)
$$

$$
{}^{Y}_{O}\vec\omega_Y = {}^{Y}C\,{}^{C}_{O}\vec\omega_C + {}^{Y}_{C}\vec\omega_Y,\qquad {}^{Y}_{C}\vec\omega_Y = \dot\alpha\,\vec e_z
$$

$$
{}^{Y}_{O}\vec a_Y = {}^{Y}C\left({}^{C}_{O}\vec a_C + {}^{C}_{O}\vec\alpha_C\times {}^{C}\vec y + {}^{C}_{O}\vec\omega_C\times {}^{C}_{O}\vec\omega_C\times {}^{C}\vec y\right)
$$

$$
{}^{Y}_{O}\vec\alpha_Y = {}^{Y}C\left({}^{C}_{O}\vec\alpha_C + {}^{C}_{O}\vec\omega_C\times {}^{Y}_{C}\vec\omega_Y\right) + {}^{Y}_{C}\vec\alpha_Y,\qquad {}^{Y}_{C}\vec\alpha_Y = \ddot\alpha\,\vec e_z
$$

> $\ddot\alpha$ 同样由 Yaw 电机角速度差分 + 低通滤波获得。

#### 步骤 2：递推云台 Pitch 的运动状态

同法令 $A=O, B=Y, Q=P$，${}^{Y}\vec p$ 是 $Y$ 原点指向 $P$ 原点的矢量，Pitch 电机角速度 ${}^{P}_{Y}\vec\omega_P=\dot\beta\,\vec e_y$：

$$
{}^{P}_{O}\vec v_P = {}^{P}Y\left({}^{Y}_{O}\vec v_Y + {}^{Y}_{O}\vec\omega_Y\times {}^{Y}\vec p\right)
$$

$$
{}^{P}_{O}\vec\omega_P = {}^{P}Y\,{}^{Y}_{O}\vec\omega_Y + \dot\beta\,\vec e_y
$$

$$
{}^{P}_{O}\vec a_P = {}^{P}Y\left({}^{Y}_{O}\vec a_Y + {}^{Y}_{O}\vec\alpha_Y\times {}^{Y}\vec p + {}^{Y}_{O}\vec\omega_Y\times {}^{Y}_{O}\vec\omega_Y\times {}^{Y}\vec p\right)
$$

$$
{}^{P}_{O}\vec\alpha_P = {}^{P}Y\left({}^{Y}_{O}\vec\alpha_Y + {}^{Y}_{O}\vec\omega_Y\times \dot\beta\,\vec e_y\right) + \ddot\beta\,\vec e_y
$$

> 注意 ${}^{P}Y = \mathrm{Pitch}^T(\beta+\gamma)$ 仍带重心偏置角 $\gamma$。

#### 步骤 3：代入 Newton/Euler 求合外力与合外扭矩

在各机构重心上分析：

**云台 Yaw**（质量近似 0 → 合外力/矩均为 0）：
$$
{}^{Y}\vec\phi_Y = m_Y\,{}^{Y}_{O}\vec a_Y = 0,\qquad
{}^{Y}\vec\tau_Y = I_Y\,{}^{Y}_{O}\vec\alpha_Y + {}^{Y}_{O}\vec\omega_Y\times I_Y\,{}^{Y}_{O}\vec\omega_Y = 0
$$

**云台 Pitch**：
$$
{}^{P}\vec\phi_P = m_P\,{}^{P}_{O}\vec a_P
$$
$$
{}^{P}\vec\tau_P = I_P\,{}^{P}_{O}\vec\alpha_P + {}^{P}_{O}\vec\omega_P\times I_P\,{}^{P}_{O}\vec\omega_P
$$

> 关键：这里的 ${}^{P}_{O}\vec a_P$ 已经**天然包含了重力**（IMU 加速度含重力，或递推时把 ${}^{C}_{O}\vec a_C$ 的重力分量带进来）。所以合外力里已含重力项 —— 这就是 5.1 说的「超集」的数学体现。

#### 步骤 4：连杆递归推到电机轴并投影

**Pitch 电机**（${}^{P}\vec y$ 为重心到 Pitch 轴的力臂）：
$$
{}^{P}\vec f_P = {}^{P}\vec\phi_P,\qquad
{}^{P}\vec n_P = {}^{P}\vec\tau_P + {}^{P}\vec y\times {}^{P}\vec\phi_P
$$
$$
n_p = {}^{P}\vec n_P^{\,T}\,\vec e_y
$$

**Yaw 电机**（把 Pitch 的力/矩转到 $Y$ 系，加上 Yaw→Pitch 力臂 ${}^{Y}\vec p$）：
$$
{}^{Y}\vec f_Y = {}^{Y}P\,{}^{P}\vec\phi_P,\qquad
{}^{Y}\vec n_Y = {}^{Y}P\,{}^{P}\vec n_P + {}^{Y}\vec p\times {}^{Y}P\,{}^{P}\vec f_P
$$
$$
n_y = {}^{Y}\vec n_Y^{\,T}\,\vec e_z
$$

$n_p, n_y$ 即为含重力+惯性的完整前馈扭矩。

> 一句话速记：**底盘运动状态逐级递推到 Yaw、Pitch → Newton/Euler 求重心处合外力矩 → 连杆递归回电机轴 → 投影。IMU 加速度自带重力，故加速度补偿一步到位。**

## 6. 三种补偿层次对比

| 维度 | 简易重力补偿 | 一般重力补偿 | 加速度补偿 |
|:----|:----|:----|:----|
| 补偿对象 | Pitch 重力矩 | Yaw+Pitch 重力矩 | Yaw+Pitch 重力+惯性力矩 |
| 核心公式 | $mgr\cos(\beta+\gamma)$ | $-m\,\vec g\times\vec r$（旋转链） | Newton-Euler 递推+递归 |
| 需要的传感器 | Pitch 编码器 | + 底盘 IMU 欧拉角 | + 底盘加/角加速度、电机角加速度 |
| 需要的标定 | $k,\gamma$ | $m, r, \gamma$ + 各原点位置 | + 惯量矩阵 $I$、各力臂矢量 |
| 计算量 | 一个 $\cos$ | 3 个 3×3 矩阵乘 + 2 叉乘 | 大量矩阵乘/叉乘，需高频运算 |
| 抗噪性 | 好（只依赖编码器） | 中（依赖 IMU 姿态） | 差（依赖二阶差分，需重滤波） |
| 适用底盘 | 平地 | 斜坡/平衡 | 高机动 |
| 工程建议 | **首选，够用** | 有爬坡需求时上 | 竞赛够用就别上，练手用 |

**误区提醒**：补偿越复杂 ≠ 效果越好。加速度补偿依赖对**角加速度**的估计，而角加速度由角速度差分得来，噪声被放大两次。若滤波没设计好，前馈项本身就在抖，反而恶化控制。**先把简易补偿做扎实，再逐层加。**

## 7. 工程落地与调参

### 7.1 前馈注入位置

前馈扭矩加在 LQR 输出**之后**、送电机**之前**。本项目的 MOTOR 模块（见 [[RM_BSP与模块教程]] §5.3）已经把这个「相加」内建到控制器里：

```c
motor->base.controller.feedforward_torque = 0.1f;  // Nm，直接加到控制输出上
```

电机的控制流是「ref + 状态反馈 → LQR 算 torque → **加 feedforward_torque** → 扭矩转电流 → CAN 发送」。所以我们**不用自己拼加法**，只需在每个控制周期把算好的重力矩写进 `feedforward_torque` 字段即可。以简易 Pitch 补偿为例（1kHz 云台控制任务）：

```c
/* gimbal_task.c —— 云台控制周期（1kHz） */

/* 标定得到的重力补偿参数（离线最小二乘拟合，见 §7.6） */
#define GRAVITY_K      0.42f    // k = m*g*r，单位 N·m
#define GRAVITY_GAMMA  0.05f    // γ 重心偏置角，单位 rad（很小可置 0）

/**
 * @brief 更新 Pitch 轴重力前馈
 * @param pitch_motor  Pitch 电机对象（内含 LQR 控制器）
 * @param cur_angle    当前 Pitch 角 β（rad），来自云台 IMU/编码器
 *
 * 注意：这里只负责「写入前馈」，LQR 反馈 + 前馈的相加、扭矩转电流、
 *      限幅与 CAN 下发都由 Motor_UpdateAll() 内部完成。
 */
void Gimbal_UpdatePitchFeedforward(Motor *pitch_motor, float cur_angle)
{
    /* 重力前馈：抵消重力矩，方向与重力矩相反 */
    float ff_gravity = -GRAVITY_K * cosf(cur_angle + GRAVITY_GAMMA);

    /* 写入控制器的前馈字段，本周期 LQR 输出会自动叠加它 */
    pitch_motor->base.controller.feedforward_torque = ff_gravity;
}

/* 控制周期主体（示意） */
void Gimbal_Task(void)
{
    float pitch_angle = INS.Pitch;                          // 当前 Pitch 角（云台 IMU）

    Motor_SetRef(&pitch_motor, target_pitch);               // 设定目标角 → LQR 的 ref
    Gimbal_UpdatePitchFeedforward(&pitch_motor, pitch_angle); // 写重力前馈

    Motor_UpdateAll();   // LQR 反馈 + feedforward_torque → 转电流 → CAN 发送
}
```

> **要点 1**：前馈是加在**电机控制器输出**（`feedforward_torque`）上，而不是改 `Motor_SetRef` 的目标角。为什么必须这样，见下面 §7.2。
>
> **要点 2**：前馈用**当前实测角度** $\beta$（`INS.Pitch`）而非目标角度算，这样云台被外力推动时前馈也跟着变，抗扰更实。
>
> **要点 3**：`feedforward_torque` 需**每周期刷新**（Pitch 角在变，重力矩就在变）。若某周期忘了写，它会残留上次的值 —— 务必放在控制周期里刷，不要只初始化一次。

#### 进阶：斜坡工况的双轴前馈（走 §4 旋转链）

上面简易版只补 Pitch、且假设底盘水平。上斜坡后按 §4：把大地重力 $(0,0,-g)$ 用旋转链转进 Y/P 系，各自算力矩、投影到电机轴，**Yaw 和 Pitch 两个电机都要写 `feedforward_torque`**。下面把 §4.3–4.5 的公式逐行翻成代码。

先备几个 3 维向量/矩阵小工具（若已链接 CMSIS-DSP 可换成 `arm_mat_*`；这里为对应公式手写）：

```c
/* ---------- 3D 向量/矩阵小工具 ---------- */
typedef float vec3[3];

/* 叉乘 out = a × b */
static void cross3(const vec3 a, const vec3 b, vec3 out) {
    out[0] = a[1]*b[2] - a[2]*b[1];
    out[1] = a[2]*b[0] - a[0]*b[2];
    out[2] = a[0]*b[1] - a[1]*b[0];
}
/* R * v */
static void mat3_mulv(const float R[3][3], const vec3 v, vec3 out) {
    for (int i = 0; i < 3; i++)
        out[i] = R[i][0]*v[0] + R[i][1]*v[1] + R[i][2]*v[2];
}
/* R^T * v （旋转矩阵求逆即转置，用于「反向」转换） */
static void mat3T_mulv(const float R[3][3], const vec3 v, vec3 out) {
    for (int i = 0; i < 3; i++)
        out[i] = R[0][i]*v[0] + R[1][i]*v[1] + R[2][i]*v[2];
}
/* 绕 z 轴旋转 α：定义为「Y 系矢量→C 系」的 Yaw(α)，即 ᶜY */
static void RotZ(float a, float R[3][3]) {
    float c = cosf(a), s = sinf(a);
    R[0][0]=c; R[0][1]=-s; R[0][2]=0;
    R[1][0]=s; R[1][1]= c; R[1][2]=0;
    R[2][0]=0; R[2][1]= 0; R[2][2]=1;
}
/* 绕 y 轴旋转 b：定义为「P 系矢量→Y 系」的 Pitch(b)，即 ʸP */
static void RotY(float b, float R[3][3]) {
    float c = cosf(b), s = sinf(b);
    R[0][0]= c; R[0][1]=0; R[0][2]=s;
    R[1][0]= 0; R[1][1]=1; R[1][2]=0;
    R[2][0]=-s; R[2][1]=0; R[2][2]=c;
}
/* ᴼC：chassis→world，由底盘 roll/pitch 建（yaw 对重力无影响，略）。
 * 约定 ᴼC = RotY(pitch)·RotX(roll)，顺序须与本项目 INS 欧拉角一致！ */
static void Rot_OC(float roll, float pitch, float R[3][3]) {
    float cr=cosf(roll), sr=sinf(roll), cp=cosf(pitch), sp=sinf(pitch);
    R[0][0]= cp;  R[0][1]= sp*sr;  R[0][2]= sp*cr;
    R[1][0]= 0;   R[1][1]= cr;     R[1][2]=-sr;
    R[2][0]=-sp;  R[2][1]= cp*sr;  R[2][2]= cp*cr;
}
```

再把 §4.3–4.5 整条链翻成一个函数（编号对应正文公式）：

```c
/* ---------- 标定/机械参数 ---------- */
#define PITCH_MASS   0.50f   // m：Pitch 固连负载质量 (kg)
#define ARM_R        0.05f   // r：Yaw 原点→Pitch 重心 距离 (m)
#define YAW_H        0.08f   // h：Yaw 原点→Chassis 原点 竖直高度 (m)
#define GRAVITY_GAMMA 0.05f  // γ：重心偏置角 (rad)
#define G_ACCEL      9.8f    // g

/**
 * @brief 斜坡工况：算 Yaw/Pitch 双轴重力前馈（对应 §4.3–4.5）
 * @param roll,pitch  底盘姿态角 (rad)，来自底盘 IMU（欧拉角约定须与本项目 INS 一致）
 * @param alpha       Yaw 电机角 α (rad)
 * @param beta        Pitch 电机角 β (rad)
 * @param[out] n_y,n_p  Yaw/Pitch 电机各自的前馈扭矩 (N·m)
 */
void Gimbal_GravityFF_Slope(float roll, float pitch, float alpha, float beta,
                            float *n_y, float *n_p)
{
    float bg = beta + GRAVITY_GAMMA;                 // β+γ
    float R_OC[3][3], R_YC_src[3][3], R_YP[3][3];
    RotZ(alpha, R_YC_src);   // ᶜY = Yaw(α)，取转置即 ʸC
    RotY(bg,    R_YP);       // ʸP = Pitch(β+γ)

    /* 由底盘 roll/pitch 建 ᴼC（chassis→world）；yaw 对重力无影响，可略 */
    Rot_OC(roll, pitch, R_OC);

    vec3 gO = {0, 0, -G_ACCEL};                      // ᴼg
    vec3 gC, gY, gP;
    mat3T_mulv(R_OC,     gO, gC);   // ᶜg = ᶜO·ᴼg = (ᴼC)ᵀ·ᴼg   (4.2 起点)
    mat3T_mulv(R_YC_src, gC, gY);   // ʸg = ʸC·ᶜg = (ᶜY)ᵀ·ᶜg   (4.2)
    mat3T_mulv(R_YP,     gY, gP);   // ᴾg = ᴾY·ʸg = (ʸP)ᵀ·ʸg   (4.1)

    /* --- Pitch 轴：ᴾn_P = −m · ᴾg × ᴾr_Y，投影到 e_y --- (4.3)(4.5) */
    vec3 rPY = {-ARM_R, 0, 0};                       // ᴾr_Y
    vec3 nP;  cross3(gP, rPY, nP);
    *n_p = -PITCH_MASS * nP[1];                      // 取 y 分量

    /* --- Yaw 轴：先拼力臂 ʸr_C = ʸP·ᴾr_Y + ʸr_C --- */
    vec3 tmp, rPC, rYC = {0, 0, -YAW_H};             // ʸr_C(段2)
    mat3_mulv(R_YP, rPY, tmp);                       // ʸP·ᴾr_Y (段1)
    for (int i = 0; i < 3; i++) rPC[i] = tmp[i] + rYC[i];

    /* ʸn_P = −m · ʸg × ʸr_C，投影到 e_z --- (4.4)(4.5) */
    vec3 nY;  cross3(gY, rPC, nY);
    *n_y = -PITCH_MASS * nY[2];                      // 取 z 分量
}
```

在控制周期里，把两个结果分别写进各自电机的前馈字段：

```c
void Gimbal_Task_Slope(void)
{
    float n_y, n_p;
    Gimbal_GravityFF_Slope(Chassis_INS.Roll, Chassis_INS.Pitch,   // 底盘姿态
                           yaw_motor.angle, pitch_motor.angle,    // 电机角
                           &n_y, &n_p);

    yaw_motor.base.controller.feedforward_torque   = n_y;         // Yaw 也要补！
    pitch_motor.base.controller.feedforward_torque = n_p;

    Motor_SetRef(&yaw_motor,   target_yaw);
    Motor_SetRef(&pitch_motor, target_pitch);
    Motor_UpdateAll();   // 各电机 LQR 反馈 + 各自 feedforward_torque → 电流 → CAN
}
```

> **和简易版的关系**：底盘水平（`roll=pitch=0`）时 `Rot_OC` 为单位阵，上面 `n_p` 退化成 $-mgr\cos(\beta+\gamma)$、`n_y` 退化成 0 —— 与 §7.1 简易代码、§4.5 一致性检查完全吻合。所以这段是简易版的超集，平地上跑它也不会错，只是多算了几个恒等旋转。
>
> **两个坑**：① Yaw 前馈**只在斜坡才非零**，平地误写非零值会让 Yaw 无故偏转 —— 务必用真实底盘姿态算，别塞常数。② 底盘姿态 `roll/pitch` 的**欧拉角约定必须和本项目 INS 输出对齐**（`Rot_OC` 的构造顺序要匹配），否则斜坡上补偿方向会反 —— 拿"云台朝斜坡上/下方时补偿是否同号"验一下。

### 7.2 前馈加在 output 还是 ref？

上面代码把前馈加在控制器输出（力矩指令）上。另一种常见接法是把前馈加到设定值（ref）上。用 LQR 云台控制举例，两者的差别关乎「前馈到底经不经过增益 $K$」。

设 LQR 反馈律 $u_{lqr} = K(x_{ref} - x)$，其中 $x=[\theta,\ \omega]^T$ 为状态，$u$ 为力矩。

**接法 A —— 前馈加到 output（推荐，也是上面代码的做法）**
$$
u = K(x_{ref} - x) + \tau_{ff}
$$
前馈力矩 $\tau_{ff}$ 直接并联到执行器指令上。

**接法 B —— 前馈加到 ref**
$$
x_{ref}' = x_{ref} + \Delta_{ff}, \qquad
u = K(x_{ref} + \Delta_{ff} - x) = K(x_{ref}-x) + K\,\Delta_{ff}
$$
前馈先改设定值，再经增益 $K$ 才到执行器。

对比最后一项：**output 前馈是 $\tau_{ff}$ 原样进电机；ref 前馈是 $K\Delta_{ff}$ 进电机 —— 中间多乘了一个 $K$。** 这带来四个实际区别：

| 维度         | 加到 output（力矩）     | 加到 ref（设定值）        |
| :--------- | :---------------- | :----------------- |
| 前馈量纲       | 必须是**力矩/电流**      | 必须是**状态量**（角度、角速度） |
| 是否过 $K$    | 1:1 直达电机，与 $K$ 解耦 | 被 $K$ 缩放，与增益强耦合    |
| 重新整定 $K$ 后 | 前馈幅度不变，稳          | 有效前馈随 $K$ 变，要重调    |
| 改变的对象      | 只改「达到目标所需的出力」     | 挪动了系统要去的目标点        |

**重力补偿为什么属于 output 侧**：

1. **量纲天然匹配**。重力矩 $mgr\cos(\beta+\gamma)$ 本身就是力矩，与电机指令同单位。要塞进 ref，得先算 $\Delta_{ff}=K^{-1}\tau_{ff}$ 反推成角度偏置；MIMO 下 $K$ 非方阵还得用伪逆，别扭且有歧义。
2. **与 $K$ 解耦**。调 LQR 的 Q/R 换了一组 $K$，重力前馈完全不用动；若走 ref，每次改 $K$ 前馈都跟着漂，标定白做。
3. **这正是 LQR 需要前馈的原因**。纯 LQR 状态反馈没有积分项，面对重力这种常值扰动必然留稳态残差（云台垂一个固定角、靠反馈力矩顶住）。把重力矩作为 output 前馈直接抵消，稳态误差归零，反馈只处理残差。

> **常见疑问：前馈让 $u(t)$ 变了，$K$ 要不要重调？—— 不用。**
> 重力是沿电机同一根轴的力矩扰动 $\tau_g$，和控制量走同一输入通道 $B$，真实动力学为 $\dot x = Ax + B(u+\tau_g)$。代入 output 前馈 $u=K(x_{ref}-x)+\tau_{ff}$：
> $$\dot x = (A-BK)\,x + BK\,x_{ref} + B(\tau_{ff}+\tau_g)$$
> 决定稳定性/阻尼/带宽的是闭环矩阵 $(A-BK)$，**里面没有 $\tau_{ff}$**；前馈只落在孤立的 $B(\tau_{ff}+\tau_g)$ 项里，令 $\tau_{ff}=-\tau_g$ 就把它消掉，系统退回标称 $\dot x=(A-BK)x+BKx_{ref}$。$K$ 决定的是 $(A-BK)$ 的极点，前馈根本不进这个矩阵 → **闭环极点不动，$K$ 不用重调**。
> 直觉：$K$ 是「状态误差 → 修正量」的策略（偏 1° 出多大力矩），不是一条 $u(t)$ 轨迹；前馈是叠加的开环项，不经反馈回路。前馈把重力这块「地形」铲平，$K$ 依旧在它本就为之设计的标称平地上工作。
> 三点补充：① 前馈标不准（$\tau_{ff}\neq-\tau_g$）也只是给反馈留点残差，仍是 $(A-BK)$ 动力学，**不失稳、$K$ 照旧**；② 唯一要防的是**执行器饱和**——前馈+反馈别一起冲过力矩上限（达妙 MIT 模式尤其留意）；③ 重力被前馈接管后，你**可以**（非必须）趁机把 LQR 调更激进（减小 $R$）拿更高带宽，这是主动优化而非前馈的要求。

**ref 前馈的正确场景**是命令/轨迹前瞻，而非扰动补偿：
- **速度前馈**：串级结构里把期望速度直接喂给速度环 ref，让位置环不必靠积累误差驱动运动（量纲是速度，匹配）。
- **轨迹前瞻**：跟随已知 $x_d(t)$ 时把未来目标提前送进 ref，让系统「领跑」不滞后。

> 判据：想抵消**已知的力/力矩扰动**（重力、摩擦、加速度惯性力）→ 加 output；想让系统**提前朝新目标动**（速度、轨迹）→ 加 ref。

> 一句话速记：**扰动补偿加 output（不过 $K$、和增益解耦）；命令前瞻加 ref（过 $K$、量纲是状态）。重力补偿是扰动补偿，永远加 output。**

### 7.3 不补的代价：稳态残差量化

在决定「到底要不要花功夫做重力补偿」之前，先把「不补会垂多少度」算成一个具体数——值不值得补，一眼就知道。

§1.2 已推出纯 LQR 的稳态下垂角：
$$
\Delta\theta = \theta_{ref}-\theta_{ss} = \frac{\tau_g}{k_1} = \frac{mgr\cos(\theta_{ref}+\gamma)}{k_1}
$$
其中 $k_1$ 是 $K=[k_1,\ k_2]$ 的**角度增益**（$u=k_1\,e_\theta+k_2\,e_\omega$，稳态 $e_\omega=0$）。

> ⚠️ **cos 里严格是实际角 $\theta_{ss}$，不是目标角 $\theta_{ref}$**。重力矩是物理量，只取决于枪管**此刻实际指向哪**，与你命令它去哪无关。所以稳态力矩平衡的严格式是一个**隐式方程**：
> $$
> k_1\,\Delta\theta = mgr\cos(\theta_{ss}+\gamma) = mgr\cos(\theta_{ref}-\Delta\theta+\gamma)
> $$
> $\Delta\theta$ 同时出现在两边、还被塞进 cos —— 超越方程，严格解需数值迭代。上式用 $\theta_{ss}\approx\theta_{ref}$ 做了**小角近似**：因为下垂角本身很小、且水平附近 cos 斜率≈0，代进去误差是二阶小量（$k_1=31.6$、$mgr=1$、水平时，近似 vs 严格误差约 0.05%，可忽略）。只有当下垂大到十几度**且**工作点接近竖直 $\pm90°$（cos 斜率大）时才需迭代——但那时 cos 本身变小、重力矩小、下垂反而小，两者互相抵消，实际系统近似基本总够用。
> 这与 §7.1 要点2「前馈用**实测角** $\beta_{measured}$ 而非目标角」是同一原理：**重力矩只认实际姿态，不管你的意图**。前馈用实测角，天然等于当前真实重力矩，不存在这个隐式偏差——这也是前馈能收敛干净的原因。

**代入一组典型数**（步兵云台量级）：设 Pitch 负载 $m=0.5\,\mathrm{kg}$、力臂 $r=0.05\,\mathrm{m}$，则水平时重力矩 $\tau_g = mgr = 0.5\times9.8\times0.05 \approx 0.245\,\mathrm{N\cdot m}$。

| 角度增益 $k_1$（N·m/rad） | 稳态下垂 $\Delta\theta = \tau_g/k_1$ | 换算成角度      |
| :------------------ | :------------------------------- | :--------- |
| 2.0                 | 0.1225 rad                       | **≈ 7.0°** |
| 5.0                 | 0.049 rad                        | **≈ 2.8°** |
| 10.0                | 0.0245 rad                       | **≈ 1.4°** |
| 20.0                | 0.01225 rad                      | **≈ 0.7°** |
|                     |                                  |            |

**怎么读这张表**：

- **下垂随 $\theta_{ref}$ 变**：$\Delta\theta \propto \cos(\theta_{ref}+\gamma)$，云台水平时下垂最大，越接近竖直越小。表中是水平（最坏）工况。
- **调大 $k_1$ 能压下垂，但不能清零**（$\tau_g/k_1$ 永远 $>0$），且代价是逼近超调、放大编码器噪声、更容易撞饱和——不能无脑加。
- **这就是「值不值得补」的判据**：自瞄要求指向精度常在 $0.5°$ 量级。上表中即使 $k_1=20$ 仍有 $0.7°$ 残差，已吃掉大半精度预算；而补了前馈后 $\Delta\theta\to 0$，$k_1$ 还能回调到更温和的值。**对自瞄步兵，补是刚需，不是锦上添花。**

> **反推 $k_1$ 的物理意义**：$k_1$ 就是云台的「回中刚度」——偏 1 rad 反馈出 $k_1$ N·m。你把上表倒过来用：实测不补时水平下垂几度，就能反算出你脚本里 $K$ 的等效 $k_1$，验证仿真和真机是否一致。

> 一句话速记：**不补时水平下垂 $\Delta\theta=mgr/k_1$，典型步兵 1~7°；调大 $k_1$ 只能压小不能清零，且引入超调/噪声。自瞄精度要 $0.5°$ → 必须补。**

### 7.4 标不准怎么办：LQI 积分增广

前馈把重力的**大头**抵消了，但标定总有误差（$k,\gamma$ 拟合偏差、负载变化、走线改动），残下一个**小扰动** $e_\tau=\tau_{ff}+\tau_g\neq 0$。§1.2/§3.4 反复强调：纯 LQR 无积分，这点残差仍会留一个（更小的）稳态角偏，比例反馈磨不掉。要**彻底清零**，给 LQR 做**积分增广（LQI）**。

**做法：把「角度误差的积分」扩进状态。** 原状态 $x=[\theta,\ \omega]^T$，新增积分状态 $\xi=\int(\theta_{ref}-\theta)\,dt$，增广状态 $\tilde x=[\theta,\ \omega,\ \xi]^T$：
$$
\dot{\tilde x} =
\underbrace{\begin{pmatrix} A & 0 \\ -C & 0 \end{pmatrix}}_{\tilde A}\tilde x
+ \underbrace{\begin{pmatrix} B \\ 0 \end{pmatrix}}_{\tilde B} u,
\qquad C=[1,\ 0]\ (\text{取出}\ \theta)
$$
对增广系统 $(\tilde A,\tilde B)$ 重新解 LQR，得 $\tilde K=[k_1,\ k_2,\ k_i]$，控制律：
$$
u = k_1(\theta_{ref}-\theta) + k_2(0-\omega) + k_i\,\xi
$$

**为什么它能清零残差**：只要还有稳态角偏，$\xi$ 就持续累积、$k_i\xi$ 不断加大，直到把常值扰动 $e_\tau$ 顶平、角偏归零为止——这正是积分作用的本质。**LQI ＝ 用「最优」的方式自动配一个积分增益 $k_i$**，省去 PID 手调 $I$ 的反复试凑。

**代价函数配置**：在 $\tilde Q=\mathrm{diag}(q_\theta,\ q_\omega,\ q_i)$ 里给积分态一个权重 $q_i$。$q_i$ 越大，消残差越快，但过大会引入积分特有的**超调和振荡**（和 PID 积分过量一个道理）。从小往上调。

> **和前馈的分工（关键，别搞反）**：**前馈干大头，LQI 兜残差**。二者不是二选一，是叠加互补：
> - 只上 LQI 不加前馈：理论上也能清零重力，但重力矩全靠积分「慢慢攒」，$\xi$ 起步阶段仍有明显下垂+滞后，动态差。
> - 只加前馈不上 LQI：下垂大幅减小，但标定残差留个小角偏。
> - **前馈 + LQI**：前馈瞬间抵消 95%+，LQI 只需消化那几个百分点的残差，$\xi$ 小、动态好、稳态零误差。这是精度要求高时的最优组合。

> **务必防积分饱和（anti-windup）**：$\xi$ 是纯积分，一旦执行器饱和（力矩撞上限）或长时间够不到目标，$\xi$ 会无限累积，退饱和后猛烈过冲。必须给 $\xi$ 限幅，或饱和时冻结积分——这点 LQI 和 PID 的 $I$ 项完全一样。

> 一句话速记：**LQI = 把角度误差积分扩进状态、重解 LQR 自动配积分增益 $k_i$，清掉纯 LQR 磨不掉的稳态残差。分工：前馈干大头、LQI 兜标定残差；记得给积分限幅防 windup。**

### 7.5 扭矩到电流的换算

达妙/DJI 电机接收的是电流（或 MIT 模式的力矩）指令。若走电流环，需用电机的转矩常数 $K_t$：

$$
I_{\text{cmd}} = \frac{\tau_{\text{cmd}}}{K_t \cdot G}
$$

其中 $G$ 为减速比。$K_t$ 查电机手册或用「堵转扭矩 / 堵转电流」实测。达妙电机 MIT 模式可直接下发力矩，省去这步 —— 参考 [[RM_BSP与模块教程]] 的 DM 电机驱动。

### 7.6 标定流程（简易补偿）

```
1. 云台上电，Pitch 进入「纯扭矩模式」（关掉 LQR 反馈，只下发恒定扭矩 q）
2. 依次设 q = 0.1, 0.2, 0.3, ... N·m
3. 每个 q 下，手扶云台缓慢松手，记录云台自然稳定的角度 β
   （稳定条件：电机扭矩 = 重力矩，即 q = -k·cos(β+γ)）
4. 收集 (q_i, β_i) 数据 ≥ 6 组，覆盖 Pitch 全行程
5. 离线用最小二乘拟合：
   令 a = k·cosγ, b = k·sinγ
   解超定方程 [−cosβ_i,  sinβ_i]·[a;b] = q_i
   还原 k = √(a²+b²), γ = atan2(b, a)
6. 把 k, γ 写回固件宏定义
```

Python 拟合脚本（离线跑，配 [[RM_LQR_lqr.py使用文档]] 一类的工具脚本）。**开环恒扭矩法和闭环 ref 扫角法采的数据可以喂进同一个函数**——因为两者的平衡点都落在 $u=mgr\cos(\theta+\gamma)$ 这同一条曲线上（见 §3.3）。核心是把非线性的 $u=k\cos(\theta+\gamma)$ 用和角公式**线性化**成对 $(a,b)$ 的线性最小二乘：

$$
u = k\cos(\theta+\gamma) = \underbrace{k\cos\gamma}_{a}\cos\theta - \underbrace{k\sin\gamma}_{b}\sin\theta
$$

##### 拟合原理：线性化 + 最小二乘在解什么

**① 为什么不能直接拟合 $k,\gamma$**：模型 $\tau=k\cos(\theta+\gamma)$ 里 $\gamma$ 藏在 $\cos$ 内部，对 $\gamma$ 是**非线性**的。硬拟合得用 Gauss-Newton/LM **迭代**——要给初值、可能陷局部解或发散，对一次性离线标定太重。

**② 和角公式把「参数非线性」变成「参数线性」**：令 $a=k\cos\gamma,\ b=k\sin\gamma$，模型变成 $\tau=a\cos\theta-b\sin\theta$。$\cos\theta,\sin\theta$ 对每个测点都是**已知常数**，未知的 $a,b$ 现在**线性**出现——非线性问题被「换参数」化成了线性问题。（通用套路：频率已知的正弦 $A\cos(\omega t+\phi)$ 都能这样拆成 $a\cos+b\sin$ 线性化。）

**③ 最小二乘 = 往列空间投影**：$N$ 个测点（$N>2$）堆成超定方程组
$$
\underbrace{\begin{bmatrix}\cos\theta_1 & -\sin\theta_1\\ \vdots & \vdots\\ \cos\theta_N & -\sin\theta_N\end{bmatrix}}_{M}\begin{bmatrix}a\\b\end{bmatrix}=\underbrace{\begin{bmatrix}\tau_1\\\vdots\\\tau_N\end{bmatrix}}_{\boldsymbol\tau}
$$
（即代码里的 `M`）。方程比未知数多、又有噪声，无精确解；退求残差平方和最小 $\min\|M\mathbf p-\boldsymbol\tau\|^2$，令偏导为零得**正规方程** $M^TM\mathbf p=M^T\boldsymbol\tau$。几何上就是把 $\boldsymbol\tau$ **垂直投影**到 $M$ 两列张成的平面上，投影点是平面内离 $\boldsymbol\tau$ 最近处——这正是「平方和最小」。`np.linalg.lstsq` 内部用 **SVD** 而非直接算 $(M^TM)^{-1}$：后者会让条件数平方、放大数值误差，SVD 更稳。

**④ 换回物理参数**：$a^2+b^2=k^2(\cos^2\gamma+\sin^2\gamma)=k^2$，故 $k=\sqrt{a^2+b^2}$（`np.hypot`）；$\gamma=\operatorname{atan2}(b,a)$——用 `atan2` 而非 `atan(b/a)` 是为了保住象限（$\gamma$ 可在 $\pm\pi$ 全程，`atan` 只能给 $\pm\tfrac{\pi}{2}$）。

**⑤ $R^2$ 判可信度**：$R^2=1-\dfrac{\sum(\tau_i-\hat\tau_i)^2}{\sum(\tau_i-\bar\tau)^2}$，分子是模型没解释掉的波动、分母是数据总波动。$R^2\to1$ 表示点贴着曲线；偏低说明噪声大或模型不对（角度用错/没停稳）。只取 $k,\gamma$ 不看 $R^2$ 是标定最容易埋的雷。

> 一句话速记：**和角公式把 $k\cos(\theta+\gamma)$ 拆成 $a\cos\theta-b\sin\theta$，参数从非线性变线性；超定方程用最小二乘（投影到列空间，SVD 求解）一步得全局最优 $(a,b)$，再 $k=\sqrt{a^2+b^2}、\gamma=\operatorname{atan2}(b,a)$ 换回；$R^2$ 判可信度。**

```python
import numpy as np

def fit_gravity(theta, tau):
    """
    最小二乘拟合重力补偿参数 k=mgr, gamma。
    theta : 实测平衡角数组 (rad)  —— 注意是【实际角】θ_ss，不是命令角！
    tau   : 对应的平衡力矩数组 (N·m) —— 云台稳态时电机的总输出力矩
    返回 : k (N·m), gamma (rad), 以及拟合质量报告 dict
    模型 : tau = k*cos(theta+gamma) = a*cos(theta) - b*sin(theta)
           其中 a = k*cos(gamma), b = k*sin(gamma)
    """
    theta = np.asarray(theta, float)
    tau   = np.asarray(tau,   float)

    # 设计矩阵 M·[a;b] = tau，对 (a,b) 线性
    M = np.column_stack([np.cos(theta), -np.sin(theta)])   # [cosθ, -sinθ]
    (a, b), *_ = np.linalg.lstsq(M, tau, rcond=None)

    k     = np.hypot(a, b)          # k = √(a²+b²)
    gamma = np.arctan2(b, a)        # γ = atan2(b, a)

    # ---- 拟合质量评估（务必看，别只取 k/gamma 就完事）----
    tau_hat = M @ np.array([a, b])          # 拟合值
    resid   = tau - tau_hat
    ss_res  = np.sum(resid**2)
    ss_tot  = np.sum((tau - tau.mean())**2)
    r2      = 1 - ss_res / ss_tot if ss_tot > 0 else float('nan')
    report  = {
        "R2":        r2,                    # 越接近 1 越好，<0.98 要警惕
        "max_abs_err": np.max(np.abs(resid)),  # 最大残差 (N·m)
        "rms_err":     np.sqrt(np.mean(resid**2)),
        "n_points":    len(theta),
    }
    return k, gamma, report

def average_friction(theta_up, tau_up, theta_dn, tau_dn):
    """
    摩擦双向平均（可选预处理，闭环扫角法用）。
    对同一组目标角分别【从下往上】和【从上往下】各逼近一次：
      - 上行停点：电机需多出力克服静摩擦 → tau 偏大
      - 下行停点：摩擦帮忙托住         → tau 偏小
    库仑摩擦方向相反、大小近似，取平均即抵消，留下纯重力矩。
    要求两趟在相同 ref 序列上采集、一一对应。
    """
    theta = 0.5 * (np.asarray(theta_up) + np.asarray(theta_dn))
    tau   = 0.5 * (np.asarray(tau_up)  + np.asarray(tau_dn))
    return theta, tau

# ---------------- 使用示例 ----------------
if __name__ == "__main__":
    # 方式A：开环恒扭矩法采的数据（theta=实测平衡角, tau=施加的恒扭矩）
    # 方式B：闭环 ref 扫角法采的数据（theta=实测稳态角, tau=电机稳态总输出力矩）
    # 两种数据结构相同，直接喂 fit_gravity 即可
    theta = np.array([-0.30, -0.15, 0.00, 0.15, 0.30, 0.45])  # rad
    tau   = np.array([ 0.28,  0.24, 0.245, 0.22, 0.19, 0.15]) # N·m

    k, gamma, rep = fit_gravity(theta, tau)
    print(f"k = {k:.4f} N·m,  gamma = {np.degrees(gamma):.2f} deg")
    print(f"拟合质量: R2={rep['R2']:.4f}, "
          f"max_err={rep['max_abs_err']:.4f} N·m, n={rep['n_points']}")
    if rep["R2"] < 0.98:
        print("⚠️ R2 偏低：检查角度是否用了实际角、是否等停稳、摩擦是否过大")
```

**用这段脚本的三条铁律**（对应 §3.3 闭环变体的三个坑）：

1. **`theta` 传实际角**：重力矩只认姿态，务必用编码器/IMU 实测的稳态角 $\theta_{ss}$，不是命令角 $\theta_{ref}$。
2. **`tau` 传总输出力矩**：直接 log 电机总力矩指令（DM MIT 模式即力矩；DJI 电流环用 $\tau=I\cdot K_t$ 换算），不要用 $k_1\times$误差反推——那需要先准确知道 $k_1$，是循环论证。
3. **看 `R2` 再收货**：$R^2<0.98$ 或 `max_abs_err` 明显大于噪声水平，说明数据有问题（没等停稳、角度用错、摩擦过大），别急着把 $k,\gamma$ 写进固件。$\gamma$ 通常只有几度（很小），若拟合出十几度以上，多半是角度零点或符号约定错了。

> **拟合完的自检**：把 $k,\gamma$ 代回 $\tau=k\cos(\theta+\gamma)$ 画一条曲线，叠上原始散点看是否贴合；水平位（$\theta=0$）的 $\tau$ 应约等于 $k\cos\gamma\approx k$，和你「称重估的 $mgr$」对得上量级才算靠谱。

### 7.7 常见问题

| 现象          | 可能原因          | 排查                       |
| :---------- | :------------ | :----------------------- |
| 补偿后云台仍缓慢下垂  | $k$ 偏小 / 符号错  | 检查 $k$ 拟合值、前馈符号（应与重力矩反向） |
| 云台上抬冲过头     | $k$ 偏大        | 重新标定，或缩一个 0.9 系数         |
| 某些角度补偿好、某些差 | $\gamma$ 没标或错 | 标定 $\gamma$，检查角度定义零点     |
| 加了加速度补偿后抖   | 角加速度差分噪声      | 加低通滤波，或降回一般/简易补偿         |
| 斜坡上仍飘       | 用了简易补偿        | 升级到一般重力补偿，引入底盘 IMU 姿态    |

#### Q：加了重力前馈，云台会不会变得「掰不动」/ 死锁在目标角？

**不会。** 一个常见误解：「LQR 命令是 0°，加了前馈后我掰 pitch 轴，它会不会一直死死顶在 0° 掰不动？」这里把两件独立的事搅在一起了——**「掰不动 / 保持角度」是反馈增益 $K$ 干的事，跟前馈无关。**

**「抓地力」来自 $K$，不是前馈。** 你现在没加前馈就已经能「命令 0° → 停在 0° → 掰它 → 顶回来」，这个顶回来的力就是反馈项：
$$
u_{fb} = K(x_{ref}-x) = K\cdot(\text{你把它掰离目标的角度})
$$
掰得越远顶得越狠，这个抓握力当前就已经存在，由 $K$ 决定。

**前馈跟误差无关，不参与和你的手较劲。** 加前馈后总控制量是：
$$
u = \underbrace{K(x_{ref}-x)}_{\text{反馈：抵抗掰动}} + \underbrace{mgr\cos(\theta+\gamma)}_{\text{前馈：只托住枪的重量}}
$$
前馈项里**没有误差 $(x_{ref}-x)$**，它不响应你掰的力，只按**当前角度**算「此刻托住这杆枪需要多大力矩」。你掰动时对抗的力**永远只有 $K\cdot$误差这一项，加前馈前后完全相同**。

> 比喻：$K$ 是把云台拉回目标的**弹簧**，前馈是**托住枪自重的一只手**。掰的时候你对抗的是弹簧；那只手只让枪不因自重下垂，不拽你。加前馈＝多了托举的手，弹簧硬度没变。

**前馈真正改的是「停得准不准」，不是「掰得动不动」：**

| | 命令 0°、静止时停在哪 | 掰它时的阻力 |
|:--|:--|:--|
| 没前馈（现状） | 停在**略低于 0°**（纯 LQR 无积分，重力拽下、$K$ 只顶回一部分，留稳态下垂） | $K\cdot$误差 |
| 加前馈 | 停在**正好 0°**（重力被前馈抵消） | $K\cdot$误差（**完全相同**） |

所以「没加前馈就能转到指定角度」完全正常——转到角度是**反馈的活**，前馈从不负责让它动起来。你系统现在能用，多半是 $K$ 够大、下垂小到没注意；加前馈是把这点下垂抹掉，顺带让反馈不必再持续输出偏置力矩托枪。**真嫌云台太硬掰不动，该调小 $K$，而不是拿掉前馈。**

> 一句话速记：**「掰不动 / 保持角度」是 $K$ 的功劳，你现在就有；前馈跟误差无关、只按角度托住重力，绝不增加抓握力。加前馈 → 停得更准（不下垂），掰的手感一点不变。嫌硬调 $K$，别怪前馈。**

> 📌 **§7 工程小结**：前馈 = 用当前角实时算的反向重力矩，加在 LQR 输出之后（本项目写进 `feedforward_torque`）；参数靠「恒扭矩测平衡角 + 最小二乘」离线标定；它只消除稳态下垂，不改反馈手感，也不用重调 $K$。

### 8.3 核心概念速查表

| 概念                          | 一句话                                        | 出现小节     |
| :-------------------------- | :----------------------------------------- | :------- |
| 前馈 vs 反馈                    | 前馈提前算干扰、反馈事后纠误差，二者相加                       | §1.1     |
| $u$ 里的重力 / 匹配扰动          | 运行时 $u$ 必含重力（纯 LQR 靠下垂 $\tau_g/k_1$ 凑）；前馈是换来源不加力 | §1.2     |
| 前馈加 output vs ref           | 扰动补偿加 output（不过 $K$）；命令前瞻加 ref（过 $K$）      | §7.2     |
| 稳态残差量化                     | 不补时水平下垂 $\Delta\theta=mgr/k_1$（典型 1~7°）；自瞄精度要 0.5° → 必须补 | §7.3     |
| LQR 无积分 / LQI              | 纯 LQR 有稳态残差；清零需积分增广（LQI：扩积分态重解 LQR）；前馈干大头、LQI 兜残差 | §1.1 §3.4 §7.4 |
| 重心偏置角 $\gamma$              | 重心连线与枪线的固定夹角，可标定可忽略                        | §3.1     |
| 标定法                         | 最小二乘拟合 $k,\gamma$；开环恒扭矩测平衡角，或闭环给 ref 扫角读力矩（只认物理平衡点 $u_{ss}=mgr\cos\theta_{ss}$，与控制器无关） | §3.3     |
| 随动随停                        | 纯前馈让云台处处中性平衡；是拟合质量的视觉验证，与 LQR 无关            | §3.3     |
| 旋转矩阵链                       | 把大地重力逐级转到云台各系                              | §4.3     |
| 叉乘矩阵形式                      | $\vec a\times = $ 反对称矩阵 $A_\times$         | §5.2.1   |
| $\dot R=\vec\omega\times R$ | 旋转矩阵对时间求导的核心恒等式                            | §5.2.2   |
| 加速度合成                       | 牵连+相对+科氏+切向+向心 五项                          | §5.2.3   |
| Newton-Euler                | 逆动力学求合外力/矩再递归回基座                           | §5.2.4-5 |
| 陀螺力矩                        | Euler 公式的 $\vec\omega\times I\vec\omega$ 项 | §5.2.5   |
