## 1. 背景：为什么需要 LQR

### 1.1 PID vs LQR

哨兵机器人的云台控制有两个核心需求：
1. **快速响应**：视觉系统检测到目标后，云台需要尽快对准
2. **无超调**：不能出现角度过冲（会导致瞄准点抖动）

传统的 PID 控制器在处理"角度+角速度"双状态系统时，往往难以同时优化响应速度和稳定性。PID 本质上是一个 SISO（单输入单输出）控制器，而云台是一个多状态系统。

**LQR（线性二次型调节器）** 是一种**最优状态反馈控制器**，它同时考虑：
- **位置误差**（角度偏离目标的代价）
- **速度**（运动过程中的能量消耗）
- **控制量**（电机输出的扭矩消耗）

通过调整权重矩阵 Q 和 R，可以在"快速响应"和"稳定无超调"之间取得最优平衡。

### 1.2 本项目中的应用

| 轴     | 状态维度      | 电机            | 增益来源              |
| ----- | --------- | ------------- | ----------------- |
| 大 Yaw | 2（角度+角速度） | GM6020        | lqr.py 计算 K_yaw   |
| 小 Yaw | 2（角度+角速度） | GM6020        | lqr.py 计算 K_yaw   |
| Pitch | 2（角度+角速度） | DM4310 (10:1) | lqr.py 计算 K_pitch |
| 底盘驱动  | 1（仅速度）    | M3508         | 手动调整              |
| 底盘转向  | 2（角度+角速度） | GM6020        | lqr.py 计算         |

## 2. 物理模型与数学原理

### 2.1 系统建模：双积分器

每个旋转轴被建模为一个**刚体旋转**系统：

```
状态向量:  x = [θ, ω]^T
            θ = 角度 (rad)
            ω = 角速度 (rad/s)

状态方程:  dx/dt = A·x + B·u

            A = [0  1]      B = [0    ]
                [0  0]          [1/J  ]

            其中 J = 转动惯量 (kg·m²)
            u = 电机输出扭矩 (N·m)
```

这是一个**双积分器模型**：角加速度 = 扭矩 / 转动惯量。忽略摩擦和弹性，适用于刚体云台的初步建模。

### 2.2 LQR 问题的数学表述

LQR 求解的是：找到一个状态反馈增益矩阵 **K**，使得控制律：

```
u = -K·x = -[K₁, K₂]·[θ, ω]^T = -K₁·θ - K₂·ω
```

最小化以下**二次代价函数**：

```
J = ∫(x^T·Q·x + u^T·R·u) dt
```

其中：
- **Q** = 状态权重矩阵（对角阵，Q₁₁ 权重角度误差，Q₂₂ 权重角速度误差）
- **R** = 控制权重标量（越大 → 控制越"省力" → 响应越慢）

### 2.3 求解算法：代数黎卡提方程

```
步骤 1: 求解连续代数黎卡提方程 (CARE):
         A^T·S + S·A - S·B·R^{-1}·B^T·S + Q = 0

步骤 2: 计算最优增益:
         K = R^{-1}·B^T·S

步骤 3: 验证闭环稳定性:
         eig(A - B·K) 的所有特征值实部 < 0

步骤 4: 仿真验证:
         odeint 求解闭环系统的阶跃响应
```

**本项目使用 `scipy.linalg.solve_continuous_are()`** 求解 CARE。

### 2.4 物理解释

- **K₁**（角度增益）：角度偏离 1 rad 时，控制器输出多少 N·m 的恢复扭矩。**K₁ 越大，回正越快**，但可能引起震荡。
- **K₂**（角速度增益）：角速度偏离 0 时，控制器输出多少 N·m 的阻尼扭矩。**K₂ 越大，阻尼越强**，可抑制震荡但响应变慢。

最优的 K₁ 和 K₂ 由 Q 和 R 唯一确定：

```
增大 Q₁₁ → K₁ 增大，K₂ 增大 → 响应更快，可能震荡
增大 Q₂₂ → K₂ 增大 → 阻尼更强，超调更小
增大 R   → K₁ 减小，K₂ 减小 → 响应更慢，更省力
```

### 3.5 脚本逻辑流程

```python
# 1. 定义物理参数
J_yaw = 0.2       # kg·m²
J_pitch = 0.04    # kg·m²
Tau_max_yaw = 2.23    # N·m
Tau_max_pitch = 8.0   # N·m

# 2. 构造状态空间模型
A = [[0, 1], [0, 0]]
B_yaw = [[0], [1/J_yaw]]
B_pitch = [[0], [1/J_pitch]]

# 3. 定义权重矩阵
Q_yaw = diag(100, 1)      # 角度权重 = 100, 角速度权重 = 1
R_yaw = 1
Q_pitch = diag(1000, 10)  # 角度权重 = 1000, 角速度权重 = 10
R_pitch = 1

# 4. 求解 LQR
K_yaw = calculate_lqr(A_yaw, B_yaw, Q_yaw, R_yaw, Tau_max_yaw, "Yaw")
K_pitch = calculate_lqr(A_pitch, B_pitch, Q_pitch, R_pitch, Tau_max_pitch, "Pitch")

# 5. 仿真并绘图
# 5 秒仿真，初始角度偏差 0.5 rad
```

## 4. 参数说明与调参指南

### 4.1 物理参数

| 参数        | Yaw 值 | Pitch 值 | 含义           | 如何获取            |
| --------- | ----- | ------- | ------------ | --------------- |
| `J`       | 0.2   | 0.04    | 转动惯量 (kg·m²) | CAD 软件计算 或 实验辨识 |
| `Tau_max` | 2.23  | 8.0     | 电机最大扭矩 (N·m) | 电机规格书           |
| `R`       | 1     | 1       | 控制权重         | 调参（见下文）         |

**转动惯量 J 的获取方法**：
1. **CAD 法**：从 SolidWorks/Fusion 360 中直接读取绕旋转轴的质量惯性矩
2. **实验辨识**：施加已知扭矩，测量角加速度，`J = τ / α`
3. **经验值**：哨兵云台的 J_yaw 通常在 0.1–0.5 之间，J_pitch 更小

### 4.2 Q 矩阵调参指南

Q = diag(q_pos, q_vel) 的唯一可调参数：

| 参数 | 作用 | 调大效果 | 调小效果 |
|------|------|---------|---------|
| `q_pos` (Q₁₁) | 惩罚角度误差 | 响应更快，回正更快，可能震荡 | 响应更慢，稳态误差更大 |
| `q_vel` (Q₂₂) | 惩罚角速度 | 阻尼更强，超调更小 | 响应可能更激进 |

### 4.3 R 参数调参指南

| 参数 | 作用 | 调大效果 | 调小效果 |
|------|------|---------|---------|
| `R` (R₁₁) | 惩罚控制量（扭矩） | 控制更"省力"，响应更慢，扭矩利用率更低 | 控制更激进，响应更快，可能饱和 |

### 4.4 调参策略

**步骤 1**：确认物理参数准确
```
- 从 CAD 获取 J 的初步值
- 确认电机最大扭矩（从规格书）
- 确认减速比（如 DM4310 有 10:1 减速器）
```

**步骤 2**：设定初始 Q 和 R
```
- 起始值：Q = diag(1, 0.1), R = 1
- 运行 lqr.py，观察阶跃响应
```

**步骤 3**：增大 q_pos 直到响应够快
```
- 每次将 q_pos 翻倍（1 → 10 → 100 → 1000）
- 观察调节时间和超调量
- 出现震荡 → 已过头，回退
```

**步骤 4**：增大 q_vel 抑制超调
```
- 如果出现超调，增大 q_vel（10 → 50 → 100）
- 目标：无超调 + 调节时间最小
```

**步骤 5**：检查扭矩利用率
```
- 输出中的 Torque Utilization 应在 50–80%
- 低于 30%：控制过于保守，可以减小 R
- 高于 90%：控制过于激进，可能在实际中不工作
```

**最终确认**：
```
- 所有特征值实部 < 0（闭环稳定）
- 无超调或超调 < 5%
- 调节时间 < 0.5 秒
- 扭矩利用率 50–80%
```

## 6. 从 Python 到 C：如何将增益部署到固件

### 6.1 完整流程

```
lqr.py
  ↓ 计算
  K_yaw = [5.47, 0.56]
  K_pitch = [38.73, 2.85]
  ↓ 手动复制
  gimbal_func.c / chassis_func.c
  ↓ 编译
  固件
  ↓ 烧录
  硬件上运行
```

### 6.2 Yaw 轴增益部署

在 `apps/sentry/gimbal_board/gimbal_func.c` 中：

**大 Yaw（GM6020，跟踪小yaw航向）**：
```c
LQR_Init_Config_s big_yaw_lqr_cfg = {
    .K = {22.36f, 3.15f},  // K₁=22.36, K₂=3.15
    .state_dim = 2,         // 角度 + 角速度
};
Motor_DJI_Init(&big_yaw_motor, &big_yaw_config);
// lqr 增益在 Motor_DJI_Init 中通过 LQRInit() 写入
```

**小 Yaw（GM6020，精确瞄准）**：
```c
LQR_Init_Config_s small_yaw_lqr_cfg = {
    .K = {5.47f, 0.56f},   // K₁=5.47, K₂=0.56
    .state_dim = 2,
};
```

**为什么大小 Yaw 的 K 值不同？**
- 大 Yaw 需要跟踪快速变化的小yaw航向（WT606 数据），需要更快的响应
- 大 Yaw 驱动的负载更重（整个上装），需要更大的增益
- 小 Yaw 只驱动弹链与发射机构，负载轻，K 值较小

### 6.3 Pitch 轴增益部署

```c
LQR_Init_Config_s pitch_lqr_cfg = {
    .K = {38.72983346f, 2.84576645f},  // 高精度值
    .state_dim = 2,
};
```

注意 pitch 轴的 K₁ 大约是 yaw 的 7 倍（38.73 vs 5.47），因为：
- J_pitch = 0.04 << J_yaw = 0.2（俯仰轴转动惯量小 5 倍）
- Q_pitch 的 q_pos = 1000 >> Q_yaw 的 q_pos = 100（俯仰对角度要求更高，惩罚力度大）
### 6.4 底盘增益部署

在 `apps/sentry/chassis_board/chassis_func.c` 中：

**驱动电机（M3508，纯速度环）**：
```c
LQR_Init_Config_s drive_lqr_cfg = {
    .K = {0.008f},     // 仅一个增益（比例速度控制）
    .state_dim = 1,    // 纯速度环
};
```

**转向电机（GM6020，角度环）**：
```c
LQR_Init_Config_s steer_lqr_cfg = {
    .K = {2.23f, 0.2f},
    .state_dim = 2,
};
```

### 6.5 C 代码中 LQR 的执行

```c
// modules/algorithm/lqr.c
float LQRCalculate(LQRInstance *lqr, float angle, float speed, float ref)
{
    if (lqr->state_dim == 2) {
        // 全状态反馈: u = -K₁*(angle - ref) - K₂*speed
        return -lqr->K[0] * (angle - ref) - lqr->K[1] * speed;
    } else {
        // 纯速度控制: u = -K₁*(speed - ref)
        return -lqr->K[0] * (speed - ref);
    }
}
```

### 6.6 固件中的额外处理

Python 模拟中不存在的实际因素：

1. **扭矩限幅**：`VAL_LIMIT(output, -max_torque, max_torque)` 防止输出超出电机能力
2. **扭矩到电流的转换**：`current = torque / (torque_constant * gear_ratio)`
3. **CAN 发送格式转换**：将 float current 映射为 int16 CAN 命令值
4. **传感器选择**：可以在电机编码器、INS 姿态、WT606 航向之间切换角度/速度反馈源

## 7. 哨兵云台与底盘的完整调参案例

### 7.1 云台 Yaw 轴调参

**初始条件**：
```
J_yaw = 0.2 kg·m²（从 CAD 获取）
Tau_max = 2.223 N·m（GM6020 规格书）
减速比 = 1:1（直驱）
```

**调参过程**：

| 迭代  | q_pos | q_vel | R   | K₁       | K₂       | 调节时间       | 超调     | 评价   |
| --- | ----- | ----- | --- | -------- | -------- | ---------- | ------ | ---- |
| 1   | 1     | 0.1   | 1   | 0.55     | 0.24     | 2.1 s      | 0%     | 太慢   |
| 2   | 10    | 0.5   | 1   | 1.73     | 0.39     | 0.7 s      | 0%     | 仍偏慢  |
| 3   | 100   | 1     | 1   | **5.47** | **0.56** | **0.15 s** | **0%** | ✅ 最优 |
| 4   | 500   | 1     | 1   | 11.2     | 0.8      | 0.08 s     | 15%    | 有过冲  |
| 5   | 500   | 10    | 1   | 11.2     | 1.2      | 0.10 s     | 3%     | 可用   |

**最终选择**：K = [5.47, 0.56]（迭代 3）。无超调 + 快速响应。

### 7.2 云台 Pitch 轴调参

**初始条件**：
```
J_pitch = 0.04 kg·m²
Tau_max = 10.0 N·m（DM4310 + 10:1 减速器）
```

**调参考量**：
- Pitch 需要比 Yaw 更高的精度（子弹的弹道受俯仰角影响更大）
- 参考值更大（q_pos = 1000 vs 100）
- 电机扭矩裕量充足（10 N·m 远大于需要的 ~6.4 N·m）

**最终选择**：K = [38.73, 2.85]，调节时间 0.07 秒，扭矩利用率 80%。

### 7.3 底盘舵轮转向调参

**目标**：控制每个舵轮的转向角，使其指向正确的方向。

**初始条件**：
```
J_steer ≈ 0.05 kg·m²（每个轮子绕转向轴）
Tau_max = 2.23 N·m（GM6020）
```

**特殊性**：转向不需要特别快的响应。关键是在不震荡的情况下保持精确指向。

**最终选择**：K = [2.23, 0.2]，较保守的增益。

### 7.4 底盘驱动速度环调参

**目标**：控制四个 M3508 驱动轮的速度。

**特殊性**：纯速度环（state_dim = 1），不使用位置反馈。

```
K = [0.008]（单一比例增益）
```

这个值通常通过实际测试手动调整，因为驱动轮的动力学很复杂（摩擦、惯量变化大）。LQR 理论计算的纯 P 增益可以作为起点。

## 8. 常见问题与排查

### Q1: 实际运行时云台震荡

**可能原因**：
1. **增益过大**：实际物理系统有未建模的延迟和弹性，Python 的理想模型过激
   - 解决：将 K₁ 和 K₂ 同时乘以 0.7–0.8，重新测试

2. **传感器噪声**：陀螺仪噪声被 LQR 放大
   - 解决：增大 K₂（加强阻尼），或在反馈路径中加入低通滤波器

3. **机械间隙**：齿轮/轴承的回差导致振荡
   - 解决：这是机械问题，LQR 无法补偿。在间隙区域增加死区

### Q2: 实际响应比仿真慢很多

**可能原因**：
1. **J 估算偏低**：实际转动惯量大于 0.2 / 0.04
   - 解决：增大 J 值后重新计算

2. **CAN 总线延迟**：电机命令通过 CAN 发送，有 1–2 ms 延迟
   - 解决：这是正常的，500 Hz 控制频率可以接受

3. **电流环响应延迟**：电机的电流环有自身的响应时间
   - 解决：在模型中增加一阶滞后环节

### Q3: 扭矩利用率始终 100%

**含义**：仿真中电机一直在全力输出。增益设置过大。

**解决**：
- 如果响应已经够快：保持，实际运行
- 如果响应过快或震荡：增大 R（如 R = 5 或 R = 10），重新计算

### Q4: J 值不确定怎么办

使用 **试凑法**：
```
1. 先猜一个 J 值（0.1–0.5 对于 yaw，0.02–0.1 对于 pitch）
2. 调出满意的 K
3. 部署到硬件测试
4. 如果震荡 → J 偏大（K 偏大），减小 J 后重新计算
5. 如果太慢 → J 偏小（K 偏小），增大 J 后重新计算
```

### Q5: Pitch 轴的 Q 为什么比 Yaw 大这么多

```
Yaw:  Q = diag(100, 1)     — 角度权重 100
Pitch: Q = diag(1000, 10)  — 角度权重 1000（10 倍）
```

原因：
1. **弹道影响**：Pitch 角误差直接影响子弹的弹道高度，每 1° 的 pitch 误差在 5 m 距离上就偏离目标约 8.7 cm
2. **力矩容量**：DM4310+10:1 有更大的扭矩余量（10 N·m），可以使用更激进的增益
3. **负载轻**：J_pitch = 0.04 << J_yaw = 0.2，对同样的 K₁，pitch 响应会更快

### Q6: 如何为新机器人调参

```
1. 从 CAD 获取所有轴的 J 值
2. 确定电机型号，记录最大扭矩和减速比
3. 设定初始 Q = diag(1, 0.1), R = 1
4. 按第 4 节的策略迭代调参
5. 在硬件上空载测试（无子弹、无装甲板碰撞）
6. 满负载测试（全弹药、快速运动）
7. 记录最终的 K 值到固件代码中
```

### Q7: Python 输出和 C 代码中的值不一致

这是正常的，因为：
- Python 计算的是理论最优值（高精度浮点）
- C 代码中的值可能经过手动调整（考虑实际硬件的非理想性）
- 如果值相差很大（>20%），请确认是否使用了错误的 J 值或最大值扭矩

建议在代码中保留注释记录之前的 K 值（如本项目中 pitch 的旧值 `{28.7312, 2.5974}` 被注释保留），便于回退和对比。
